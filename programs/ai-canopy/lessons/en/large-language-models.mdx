## What Is an LLM? 🤖

A **Large Language Model** (LLM) is a neural network trained on massive amounts of text to understand and generate human language. The word "large" refers to three things:

```
Scale of modern LLMs:
┌──────────────────────┬──────────────────────────────────┐
│ Parameters           │ Billions (7B → 400B+)            │
│ Training data        │ Trillions of tokens (words)      │
│ Training cost        │ Millions of dollars               │
│ Training time        │ Weeks to months on thousands of GPUs │
└──────────────────────┴──────────────────────────────────┘
```

At its core, an LLM does one thing: **predict the next token**. Given "The cat sat on the", it predicts "mat" (or "roof" or "couch") with different probabilities. This simple objective, at massive scale, produces remarkably intelligent behaviour.

<ThinkAboutIt>
Imagine reading every book, article, and website ever written — billions of pages. After all that reading, you'd be pretty good at predicting what word comes next in any sentence. That's essentially what an LLM does, but with mathematical precision.
</ThinkAboutIt>

## The Transformer Architecture 🏗️

Every modern LLM is built on the **Transformer** architecture (from the 2017 paper "Attention Is All You Need"). The key innovation: **self-attention**.

### Self-Attention — The Core Idea

Traditional models read text sequentially — word by word, left to right. Transformers read **everything at once** and figure out which words are relevant to each other.

```
Sentence: "The bank by the river was steep"

For the word "bank", attention scores might be:
  "bank" ←→ "river"  = 0.45  (high — helps clarify meaning)
  "bank" ←→ "steep"  = 0.30  (medium — supports the "riverbank" meaning)
  "bank" ←→ "The"    = 0.05  (low — not very informative)
```

This lets the model understand context: "bank" near "river" means a riverbank, not a financial bank.

### Multi-Head Attention

Transformers have **multiple attention heads** running in parallel, each learning different relationships (grammar, meaning, coreference). GPT-3 has 96 heads across 96 layers — all self-discovered, no human guidance.

Each Transformer layer follows: **Self-Attention → Add + Normalise → Feed-Forward → Add + Normalise**. Stack 50–100+ of these blocks and you have a modern LLM.

<FunFact>
The "Add + Normalise" steps are skip connections — the same trick from ResNet! They keep gradients healthy across dozens of layers, making deep Transformers trainable.
</FunFact>

## The Training Pipeline 🔄

Training an LLM happens in three distinct phases:

### Phase 1: Pretraining (learn language)

The model reads trillions of tokens from books, websites, and code. It learns grammar, facts, reasoning patterns, and even some world knowledge — all from predicting the next token.

```
Input:  "The capital of France is ___"
Target: "Paris"

Input:  "def fibonacci(n):\n    if n <= 1:\n        return ___"
Target: "n"
```

**Cost:** Millions of dollars and weeks of GPU time. This is the expensive step.

### Phase 2: Fine-Tuning (learn to follow instructions)

The base model is great at completing text but terrible at following instructions. Fine-tuning trains it on curated question-answer pairs:

```
User: "Summarise this article in three bullet points."
Assistant: "• Point one...\n• Point two...\n• Point three..."
```

### Phase 3: RLHF (learn human preferences)

**Reinforcement Learning from Human Feedback** teaches the model what humans consider helpful, harmless, and honest.

```
Prompt: "How do I pick a lock?"

Response A: "Here are detailed instructions..."   ← Ranked lower
Response B: "I can't help with that because..."   ← Ranked higher

The model learns: safety and helpfulness matter.
```

<Callout>
RLHF is what makes the difference between a model that just completes text and one that feels like a helpful assistant. It aligns the model with human values — but it's not perfect, which is why AI safety research remains critical.
</Callout>

## Capabilities and Limitations ⚖️

### What LLMs Can Do Well
- Generate fluent, coherent text in many languages
- Summarise, translate, and transform text
- Write and explain code
- Answer questions using knowledge from training data
- Reason through multi-step problems (with the right prompting)

### What LLMs Struggle With
- **Hallucinations** — confidently stating incorrect facts
- **Maths** — unreliable with complex arithmetic without tools
- **Recency** — knowledge has a training cutoff date
- **True reasoning** — pattern-matching can look like reasoning but fail on novel problems
- **Privacy** — they may memorise and regurgitate training data

<ThinkAboutIt>
LLMs are like brilliant but unreliable interns. They can draft amazing work, but you should always fact-check their output. Trust, but verify — especially for anything critical.
</ThinkAboutIt>

## Token Economics and Context Windows 📊

LLMs don't read characters or words — they read **tokens**. A token is roughly 3–4 characters or about ¾ of a word.

```
"Hello, how are you today?" → ["Hello", ",", " how", " are", " you", " today", "?"]
                               = 7 tokens

Rule of thumb:
  100 tokens  ≈  75 words
  1,000 tokens ≈  750 words  ≈  1.5 pages
```

The **context window** is the maximum tokens an LLM can process at once (input + output). GPT-4o supports 128K tokens (~300 pages), Claude handles 200K (~500 pages). API pricing is per token, so understanding token economics is essential:

```python
# Rough cost estimation
input_tokens = 1000    # Your prompt
output_tokens = 500    # Model's response
price_per_1k = 0.01   # Varies by model and provider

cost = ((input_tokens + output_tokens) / 1000) * price_per_1k
print(f"Cost per request: ${cost:.4f}")   # $0.0150
print(f"Cost for 10,000 requests: ${cost * 10000:.2f}")  # $150.00
```

## Hands-On: Using an LLM API 🛠️

Here's how to call an LLM API in Python. This pattern works similarly across providers:

```python
from openai import OpenAI

# Initialise the client (API key from environment variable)
client = OpenAI()

# Send a request
response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {
            "role": "system",
            "content": "You are a helpful science tutor for teenagers."
        },
        {
            "role": "user",
            "content": "Explain photosynthesis in simple terms."
        }
    ],
    temperature=0.7,    # 0 = deterministic, 1 = creative
    max_tokens=300       # Limit response length
)

# Extract the reply
answer = response.choices[0].message.content
print(answer)

# Check token usage
usage = response.usage
print(f"Input tokens:  {usage.prompt_tokens}")
print(f"Output tokens: {usage.completion_tokens}")
print(f"Total tokens:  {usage.total_tokens}")
```

**Key parameters explained:**

```
temperature:  Controls randomness (0 = focused, 1 = creative)
max_tokens:   Limits response length (saves cost)
messages:     The conversation history (system + user + assistant turns)
model:        Which LLM to use
```

<Callout>
The `messages` array is your conversation history. The model doesn't "remember" previous conversations — you send the full context every time. This is why context windows matter: longer conversations cost more tokens.
</Callout>

## Quick Recap 🎯

1. **LLMs** are neural networks trained on trillions of tokens to predict the next word — scale creates emergent intelligence
2. **Transformers** use self-attention to understand context, processing all tokens in parallel
3. **Training pipeline**: pretraining (learn language) → fine-tuning (follow instructions) → RLHF (align with human values)
4. **Major models** (GPT, Claude, Llama, Gemini) each have different strengths — no single "best" model
5. LLMs are powerful but **not perfect** — hallucinations, maths errors, and knowledge cutoffs are real limitations
6. **Tokens** are the currency of LLMs — understanding them helps you manage cost and context
