# Attention and Transformers — The Architecture That Changed Everything

Every major language model today — GPT-4, Claude, Gemini, LLaMA — is built on the Transformer architecture. Published in the landmark 2017 paper *Attention Is All You Need*, it replaced recurrence with a mechanism called **self-attention**, unlocking unprecedented parallelism and performance.

## Why RNNs Hit a Wall

Recurrent Neural Networks process sequences one token at a time, carrying a hidden state forward. This creates two crippling problems:

- **Vanishing gradients** — By the time a gradient propagates back through hundreds of time steps, it shrinks to near zero, making it impossible to learn long-range dependencies.
- **Sequential bottleneck** — Each step depends on the previous one, so you cannot parallelise training across a sequence. A 1,000-token input requires 1,000 serial operations.

LSTMs and GRUs added gating mechanisms that eased vanishing gradients, but they never solved the parallelisation problem. Training remained fundamentally slow.

## The Attention Intuition

Imagine reading a long paragraph and being asked *"What did the CEO announce?"* Your eyes don't re-read every word sequentially — they **jump to the relevant sentence**. Attention works the same way: it lets each token look at every other token and decide which ones matter most for its current context.

Think of it as a **spotlight on a stage**. Instead of illuminating the entire stage uniformly, the model learns to shine a bright beam on the tokens that carry the most relevant information for the task at hand.

## Query, Key, Value — The Core Mechanism

Self-attention borrows terminology from information retrieval. For each token, the model computes three vectors:

- **Query (Q)** — *"What am I looking for?"*
- **Key (K)** — *"What information do I contain?"*
- **Value (V)** — *"What do I actually contribute if selected?"*

The attention score between two tokens is the **dot product of Q and K**, scaled by √d_k (the key dimension) to prevent exploding values. Applying softmax yields normalised weights, and the output is a weighted sum of all V vectors:

```
Attention(Q, K, V) = softmax(Q·Kᵀ / √d_k) · V
```

**Walkthrough:** Consider the sentence *"The cat sat on the mat because it was tired."* When processing *"it"*, the query for *"it"* attends most strongly to *"cat"* (high Q·K score), resolving the ambiguous pronoun — no recurrence needed.

## Multi-Head Attention

A single attention head can only capture one type of relationship at a time. **Multi-head attention** runs several attention heads in parallel, each with its own learned Q, K, V projections, then concatenates and projects the results:

```
MultiHead(Q, K, V) = Concat(head₁, ..., headₕ) · Wᴼ
```

Different heads learn different linguistic patterns — one might attend to syntactic dependencies, another to semantic similarity, and a third to positional proximity. GPT-3 uses 96 heads; each sees a 128-dimensional slice of the full 12,288-dimensional space.

<FunFact>
Researchers at Google discovered that individual attention heads in BERT specialise remarkably — some track subject-verb agreement, others handle coreference resolution, and some even learn dependency parsing, all without explicit supervision.
</FunFact>

## Positional Encoding

Self-attention is **permutation-invariant** — it treats *"dog bites man"* identically to *"man bites dog"* unless we inject positional information. The original Transformer used sinusoidal encodings:

```
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

Each position gets a unique pattern of sine and cosine waves at different frequencies. This allows the model to learn relative positions because `PE(pos+k)` can be expressed as a linear function of `PE(pos)`. Modern models (GPT, LLaMA) use **Rotary Positional Embeddings (RoPE)**, which encode relative position directly into the Q/K dot product.

## The Full Transformer Architecture

The original Transformer has an **encoder-decoder** structure:

- **Encoder** — Processes the full input with bidirectional self-attention (every token attends to every other). Stacks N layers of self-attention + feed-forward blocks with residual connections and layer normalisation.
- **Decoder** — Generates output auto-regressively with **causal (masked) self-attention** (each token can only attend to previous tokens) plus cross-attention to the encoder output.

Each sub-layer follows the pattern: `LayerNorm(x + Sublayer(x))`, where the residual connection ensures gradients flow freely through deep stacks.

## Why Transformers Parallelise

The key insight: self-attention computes all pairwise token interactions in a **single matrix multiplication**. For a sequence of length n, the entire n×n attention matrix is computed simultaneously on GPU hardware. Contrast this with RNNs, which require n sequential steps regardless of available compute.

This O(1) depth (constant sequential operations) versus the RNN's O(n) depth is precisely why Transformers scale to sequences of thousands of tokens during training.

## Decoder-Only vs Encoder-Only vs Encoder-Decoder

Three architectural variants emerged from the original design:

| Variant | Attention | Example | Best For |
|---------|-----------|---------|----------|
| **Encoder-only** | Bidirectional | BERT, RoBERTa | Classification, NER, embeddings |
| **Decoder-only** | Causal (left-to-right) | GPT, LLaMA, Claude | Text generation, chat, reasoning |
| **Encoder-decoder** | Both | T5, BART, Whisper | Translation, summarisation |

Decoder-only models dominate today because **generation is the universal interface** — any NLP task can be framed as text generation. This architectural simplicity also scales more predictably.
