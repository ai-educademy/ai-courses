# RLHF और Alignment — ChatGPT कैसे सीखता है Human Values

एक pre-trained language model एक powerful लेकिन uncontrolled engine है। यह poetry लिख सकता है, code generate कर सकता है, physics पर reason कर सकता है — लेकिन toxic content भी produce कर सकता है, confidently facts fabricate कर सकता है, और user instructions ignore कर सकता है। **Alignment** वह process है जिसमें इस raw capability को genuinely helpful, honest और harmless बनने की दिशा में steer किया जाता है।

## Alignment Problem

Pre-training एक single objective optimise करती है: **next token predict करो**। इससे model को broad knowledge मिलती है लेकिन यह समझ नहीं आती कि human actually *क्या चाहता है*। Base model से सवाल पूछो और वह text को ऐसे complete कर सकता है जैसे forum post लिख रहा हो, advertisement हो, या conspiracy theory — क्योंकि ये सभी patterns उसके training data में हैं।

"Capable" और "useful" के बीच का gap alignment problem है। जो model medical advice लिख सकता है लेकिन refuse करना नहीं जानता, वह potentially dangerous है। इस gap को पाटने के लिए model को **human preferences** सिखानी होंगी — सिर्फ यह नहीं कि आगे क्या *आ सकता* है, बल्कि यह कि आगे क्या *आना चाहिए*।

## Stage 1: Supervised Fine-Tuning (SFT)

पहला step straightforward है: हज़ारों high-quality **demonstration examples** collect करो — user और assistant के बीच ideal conversations — और base model को उन पर fine-tune करो। इससे model helpful होने का *format* सीखता है: सवालों का directly जवाब देना, instructions follow करना, uncertainty acknowledge करना, और harmful requests refuse करना।

SFT अकेला noticeably better model produce करता है, लेकिन इसकी limitations हैं। Human demonstrators सीमित examples ही लिख सकते हैं, और model उन demonstrations की *average* quality imitate करना सीखता है, consistently best possible response produce करने की बजाय।

## Stage 2: Reward Model Training

Ideal responses लिखने की बजाय, human labellers को **model outputs के pairs** दिए जाते हैं और पूछा जाता है कौन सा बेहतर है। यह cognitively आसान है — compare करना create करने से simpler है — और काफ़ी ज़्यादा preference data collect करने देता है।

ये ranked pairs एक **reward model (RM)** train करते हैं: एक अलग neural network जो prompt और response input लेता है और human preference represent करने वाला scalar score output करता है। RM ऐसे patterns सीखता है:

- Accurate, sourced responses vague ones से higher score पाते हैं
- Uncertainty acknowledge करने वाले responses confident fabrications से higher score पाते हैं
- Helpful refusals harmful requests comply करने से higher score पाते हैं

## Stage 3: Reinforcement Learning (PPO)

Reward model तैयार होने पर, SFT model को **Proximal Policy Optimisation (PPO)** से further optimise किया जाता है:

1. Model एक prompt पर response generate करता है
2. Reward model response score करता है
3. PPO model weights update करता है ताकि expected reward बढ़े
4. **KL divergence penalty** model को SFT baseline से बहुत दूर drift होने से रोकती है (reward hacking avoid करने के लिए)

KL penalty critical है। इसके बिना, model reward model की quirks exploit करना सीख जाएगा — ऐसे responses generate करेगा जो highly score होते हैं लेकिन nonsensical या repetitive होते हैं। Policy को SFT model से कितना diverge हो सकती है यह constrain करके, PPO alignment improve करते हुए outputs coherent रखता है।

यह तीन-stage process — SFT → Reward Model → PPO — बिल्कुल वही है जिससे OpenAI ने GPT-3.5 को ChatGPT में transform किया, जैसा उनके **InstructGPT** paper में describe है।

<FunFact>
InstructGPT paper ने एक striking result reveal किया: RLHF से aligned 1.3B parameter model को human judges ने unaligned 175B GPT-3 — 100× बड़े model — के ऊपर prefer किया। Alignment raw scale से ज़्यादा matter कर सकती है।
</FunFact>

## DPO: बिचौलिया हटाना

**Direct Preference Optimisation (DPO)** RLHF pipeline को simplify करता है separate reward model पूरी तरह eliminate करके। DPO alignment को classification problem के रूप में frame करता है: preferred response और rejected response दिए जाने पर, language model के weights directly update करो ताकि preferred response की probability बढ़े।

DPO loss function implicitly language model के अंदर ही reward model define करता है:

```
L_DPO = -log σ(β · (log π(y_w|x)/π_ref(y_w|x) - log π(y_l|x)/π_ref(y_l|x)))
```

जहाँ `y_w` preferred response है, `y_l` rejected है, और `π_ref` reference (SFT) model है। DPO simpler, ज़्यादा stable, और train करने में सस्ता है — कोई reward model नहीं, कोई RL loop नहीं, कोई hyperparameter-sensitive PPO नहीं। Meta का LLaMA 2 और कई open-source models अब DPO या इसके variants use करते हैं।

<ThinkAboutIt>
DPO reward model eliminate करता है लेकिन inference time पर arbitrary new responses score करने की ability भी खो देता है। किन scenarios में explicit reward model (full RLHF pipeline) रखना DPO की simplicity से ज़्यादा valuable हो सकता है?
</ThinkAboutIt>

## Constitutional AI: Scale पर Self-Supervision

Anthropic ने **Constitutional AI (CAI)** introduce किया ताकि human labellers पर निर्भरता कम हो। Approach:

1. Principles का set define करो (एक "constitution") — जैसे *"Choose the response that is most helpful while being honest and harmless"*
2. Response pairs generate करो और **model खुद** judge करे कि कौन सा response constitution को बेहतर follow करता है
3. इन AI-generated preferences से reward model train करो (RLAIF — RL from AI Feedback)

CAI preference collection dramatically scale करता है — AI millions of comparisons human labelling के fraction cost पर generate और evaluate कर सकता है। यह alignment criteria को **explicit और auditable** भी बनाता है।

## Red Teaming

कोई alignment technique perfect नहीं है। **Red teaming** में deliberately model से harmful, biased, या untruthful outputs produce करवाने की कोशिश की जाती है। Red teamers probe करते हैं:

- Safety training bypass करने वाले jailbreaks
- Demographics के across responses में systematic biases
- High-stakes domains (medical, legal) में factual errors
- Refusal boundaries में inconsistencies

Major labs अब models release करने से पहले extensive red teaming conduct करते हैं, अक्सर external security researchers और domain experts को involve करते हुए।

## Alignment Tax

Alignment एक cost लेकर आती है। RLHF-trained models कभी-कभी **ज़रूरत से ज़्यादा cautious** हो जाते हैं — benign requests refuse करते हैं, unnecessary caveats add करते हैं, या perceived risk avoid करने के लिए bland responses produce करते हैं। यह "alignment tax" एक real engineering challenge है:

- बहुत कम alignment → harmful outputs
- बहुत ज़्यादा alignment → unhelpful, annoying refusals

Alignment research का frontier ऐसी techniques ढूँढना है जो genuine safety boundaries maintain करते हुए helpfulness maximise करें।

<FunFact>
OpenAI ने GPT-4 के base model को public release version में transform करने के लिए alignment work (SFT data collection, reward modelling, और RLHF training) पर लगभग छह महीने खर्च किए। Pre-training compute का fraction होते हुए भी alignment phase को हज़ारों घंटों की specialised human labelling चाहिए थी।
</FunFact>
