# Attention మరియు Transformers - ప్రతిదీ మార్చిన Architecture

నేడు ప్రతి major language model - GPT-4, Claude, Gemini, LLaMA - Transformer architecture పై build అయింది. 2017 landmark paper *Attention Is All You Need* లో publish అయిన ఇది recurrence ని **self-attention** mechanism తో replace చేసి, unprecedented parallelism మరియు performance unlock చేసింది.

## RNNs ఎందుకు గోడకు తగిలాయి

Recurrent Neural Networks sequences ని ఒక token చొప్పున process చేస్తాయి, hidden state ని forward carry చేస్తాయి. ఇది రెండు crippling problems create చేస్తుంది:

- **Vanishing gradients** - Gradient hundreds of time steps ద్వారా back propagate అయ్యేసరికి, దాదాపు zero కి shrink అవుతుంది, long-range dependencies నేర్చుకోవడం impossible అవుతుంది.
- **Sequential bottleneck** - ప్రతి step ముందు step పై depend అవుతుంది, కాబట్టి sequence అంతటా training parallelise చేయలేరు. 1,000-token input కి 1,000 serial operations అవసరం.

LSTMs మరియు GRUs gating mechanisms add చేసి vanishing gradients ease చేశాయి, కానీ parallelisation problem ఎప్పుడూ solve చేయలేదు. Training fundamentally slow గానే ఉంది.

## Attention Intuition

ఒక పెద్ద paragraph చదివి *"CEO ఏమి announce చేశారు?"* అని అడిగారనుకోండి. మీ కళ్ళు ప్రతి word ని sequentially re-read చేయవు - **relevant sentence కి jump చేస్తాయి**. Attention అదే విధంగా work చేస్తుంది: ప్రతి token ప్రతి ఇతర token ని చూసి ప్రస్తుత context కి ఏవి most matter అవుతాయో decide చేయగలదు.

దీన్ని **stage పై spotlight** గా think చేయండి. Entire stage ని uniformly illuminate చేయడం బదులు, model task కి most relevant information carry చేసే tokens పై bright beam shine చేయడం నేర్చుకుంటుంది.

## Query, Key, Value - Core Mechanism

Self-attention information retrieval నుండి terminology borrow చేస్తుంది. ప్రతి token కి model మూడు vectors compute చేస్తుంది:

- **Query (Q)** - *"నేను ఏమి search చేస్తున్నాను?"*
- **Key (K)** - *"నా దగ్గర ఏ information ఉంది?"*
- **Value (V)** - *"Select అయితే నేను నిజంగా ఏమి contribute చేస్తాను?"*

రెండు tokens మధ్య attention score **Q మరియు K యొక్క dot product**, √d_k (key dimension) తో scale చేయబడుతుంది - exploding values prevent చేయడానికి. Softmax apply చేస్తే normalised weights వస్తాయి, output అన్ని V vectors యొక్క weighted sum:

```
Attention(Q, K, V) = softmax(Q·Kᵀ / √d_k) · V
```

**ఉదాహరణ:** *"The cat sat on the mat because it was tired."* sentence consider చేయండి. *"it"* process చేసినప్పుడు, *"it"* query *"cat"* కి most strongly attend చేస్తుంది (high Q·K score), ambiguous pronoun ని resolve చేస్తుంది - recurrence అవసరం లేదు.

## Multi-Head Attention

Single attention head ఒక సమయంలో ఒక type relationship మాత్రమే capture చేయగలదు. **Multi-head attention** parallel లో multiple attention heads run చేస్తుంది, ప్రతి దాని own learned Q, K, V projections తో, తర్వాత results concatenate మరియు project చేస్తుంది:

```
MultiHead(Q, K, V) = Concat(head₁, ..., headₕ) · Wᴼ
```

Different heads different linguistic patterns నేర్చుకుంటాయి - ఒకటి syntactic dependencies attend చేయవచ్చు, మరొకటి semantic similarity, ఇంకొకటి positional proximity. GPT-3 96 heads use చేస్తుంది; ప్రతి ఒక్కటి full 12,288-dimensional space లో 128-dimensional slice చూస్తుంది.

<FunFact>
Google researchers discover చేశారు: BERT లో individual attention heads remarkably specialise అవుతాయి - కొన్ని subject-verb agreement track చేస్తాయి, ఇతరవి coreference resolution handle చేస్తాయి, కొన్ని dependency parsing కూడా నేర్చుకుంటాయి, explicit supervision లేకుండానే.
</FunFact>

## Positional Encoding

Self-attention **permutation-invariant** - positional information inject చేయకపోతే *"dog bites man"* ని *"man bites dog"* గా treat చేస్తుంది. Original Transformer sinusoidal encodings use చేసింది:

```
PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
```

ప్రతి position కి different frequencies లో sine మరియు cosine waves unique pattern ఉంటుంది. `PE(pos+k)` ని `PE(pos)` యొక్క linear function గా express చేయగలరు కాబట్టి model relative positions నేర్చుకోగలదు. Modern models (GPT, LLaMA) **Rotary Positional Embeddings (RoPE)** use చేస్తాయి, relative position ని Q/K dot product లో directly encode చేస్తాయి.

## పూర్తి Transformer Architecture

Original Transformer **encoder-decoder** structure కలిగి ఉంటుంది:

- **Encoder** - Full input ని bidirectional self-attention తో process చేస్తుంది (ప్రతి token ప్రతి ఇతర token కి attend చేస్తుంది). Self-attention + feed-forward blocks N layers stack చేస్తుంది, residual connections మరియు layer normalisation తో.
- **Decoder** - **Causal (masked) self-attention** తో auto-regressively output generate చేస్తుంది (ప్రతి token previous tokens కి మాత్రమే attend చేయగలదు), encoder output కి cross-attention తో కలిపి.

ప్రతి sub-layer pattern: `LayerNorm(x + Sublayer(x))`, residual connection deep stacks ద్వారా gradients freely flow అవడానికి ensure చేస్తుంది.

<ThinkAboutIt>
Transformer residual connections gradients ని 100+ layers ద్వారా directly flow అవడానికి allow చేస్తాయి. ఇవి లేకపోతే 96-layer GPT-3 train చేయడం దాదాపు impossible. Deep computer vision కి ResNets ని viable చేసిన skip connections తో ఇది ఎలా relate అవుతుంది?
</ThinkAboutIt>

## Transformers ఎందుకు Parallelise అవుతాయి

Key insight: self-attention అన్ని pairwise token interactions ని **ఒకే matrix multiplication** లో compute చేస్తుంది. Length n sequence కి, entire n×n attention matrix GPU hardware పై simultaneously compute అవుతుంది. RNNs తో contrast చేస్తే, available compute తో సంబంధం లేకుండా n sequential steps అవసరం.

Training సమయంలో Transformers thousands of tokens sequences కి scale అవడానికి RNN O(n) depth కంటే ఈ O(1) depth (constant sequential operations) ఖచ్చితంగా కారణం.

## Decoder-Only vs Encoder-Only vs Encoder-Decoder

Original design నుండి మూడు architectural variants emerge అయ్యాయి:

| Variant | Attention | Example | Best For |
|---------|-----------|---------|----------|
| **Encoder-only** | Bidirectional | BERT, RoBERTa | Classification, NER, embeddings |
| **Decoder-only** | Causal (left-to-right) | GPT, LLaMA, Claude | Text generation, chat, reasoning |
| **Encoder-decoder** | Both | T5, BART, Whisper | Translation, summarisation |

Decoder-only models నేడు dominate చేస్తున్నాయి ఎందుకంటే **generation universal interface** - ఏ NLP task నైనా text generation గా frame చేయవచ్చు. ఈ architectural simplicity more predictably scale అవుతుంది.
