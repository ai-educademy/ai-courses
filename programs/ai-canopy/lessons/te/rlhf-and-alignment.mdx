# RLHF మరియు Alignment - ChatGPT మానవ విలువలు ఎలా నేర్చుకుంటుంది

Pre-trained language model ఒక powerful కానీ uncontrolled engine. Poetry రాయగలదు, code generate చేయగలదు, physics గురించి reason చేయగలదు - కానీ toxic content produce చేయగలదు, confidently facts fabricate చేయగలదు, user instructions ignore చేయగలదు. **Alignment** అనేది ఆ raw capability ని genuinely helpful, honest, మరియు harmless వైపు steer చేసే process.

## Alignment Problem

Pre-training ఒకే objective optimise చేస్తుంది: **next token predict చేయడం**. ఇది model కి broad knowledge ఇస్తుంది కానీ human *actually* ఏమి కోరుకుంటున్నారో అనే understanding ఇవ్వదు. Base model ని question అడిగితే forum post, advertisement, లేదా conspiracy theory రాస్తున్నట్లు text complete చేయవచ్చు - ఎందుకంటే ఈ patterns అన్నీ training data లో ఉన్నాయి.

"Capable" మరియు "useful" మధ్య gap - ఇదే alignment problem. Medical advice రాయగల కానీ ఎప్పుడు refuse చేయాలో తెలియని model potentially dangerous. ఈ gap close చేయడానికి model కి **human preferences** నేర్పించాలి - ఏది *next* రాగలదో కాదు, ఏది *next* రావాలో.

## Stage 1: Supervised Fine-Tuning (SFT)

మొదటి step straightforward: వేల high-quality **demonstration examples** - user మరియు assistant మధ్య ideal conversations - collect చేసి, base model ని వాటి పై fine-tune చేయడం. ఇది model కి helpful గా ఉండే *format* నేర్పిస్తుంది: questions directly answer చేయడం, instructions follow చేయడం, uncertainty admit చేయడం, harmful requests refuse చేయడం.

SFT alone noticeably better model produce చేస్తుంది, కానీ limitations ఉన్నాయి. Human demonstrators limited examples మాత్రమే రాయగలరు, model consistently best possible response produce చేయడం బదులు demonstrations *average* quality imitate చేయడం నేర్చుకుంటుంది.

## Stage 2: Reward Model Training

Ideal responses రాయడం బదులు, human labellers కి **model outputs pairs** ఇచ్చి ఏది better అని rank చేయమని అడుగుతారు. Compare చేయడం create చేయడం కంటే cognitively easier - far more preference data collect చేయగలరు.

ఈ ranked pairs **reward model (RM)** train చేస్తాయి: prompt మరియు response input తీసుకుని human preference represent చేసే scalar score output ఇచ్చే separate neural network. RM ఇలాంటి patterns నేర్చుకుంటుంది:

- Accurate, sourced responses vague responses కంటే higher score
- Uncertainty acknowledge చేసే responses confident fabrications కంటే higher score
- Helpful refusals harmful requests comply చేయడం కంటే higher score

## Stage 3: Reinforcement Learning (PPO)

Reward model ready అయిన తర్వాత, SFT model ని **Proximal Policy Optimisation (PPO)** ఉపయోగించి further optimise చేస్తారు. Process:

1. Model prompt కి response generate చేస్తుంది
2. Reward model response ని score చేస్తుంది
3. PPO expected reward increase చేయడానికి model weights update చేస్తుంది
4. **KL divergence penalty** model SFT baseline నుండి too far drift అవకుండా prevent చేస్తుంది (reward hacking avoid చేయడం)

KL penalty critical. ఇది లేకపోతే, model reward model quirks exploit చేయడం నేర్చుకుంటుంది - highly score అయ్యే కానీ nonsensical లేదా repetitive responses generate చేయడం. Policy SFT model నుండి ఎంత deviate అవగలదో constrain చేయడం ద్వారా, PPO alignment improve చేస్తూ outputs coherent గా ఉంచుతుంది.

ఈ three-stage process - SFT → Reward Model → PPO - OpenAI GPT-3.5 ని ChatGPT గా transform చేసిన విధానం, **InstructGPT** paper లో describe చేసినట్లు.

<FunFact>
InstructGPT paper striking result reveal చేసింది: RLHF తో aligned 1.3B parameter model ని human judges unaligned 175B GPT-3 కంటే prefer చేశారు - 100× larger model. Alignment raw scale కంటే more matter అవుతుందని నిరూపించింది.
</FunFact>

## DPO: Middleman ని తొలగించడం

**Direct Preference Optimisation (DPO)** separate reward model ని పూర్తిగా eliminate చేసి RLHF pipeline simplify చేస్తుంది. DPO alignment ని classification problem గా frame చేస్తుంది: preferred response మరియు rejected response ఇచ్చి, preferred response probability increase చేయడానికి language model weights directly update చేయడం.

DPO loss function language model లోనే implicitly reward model define చేస్తుంది:

```
L_DPO = -log σ(β · (log π(y_w|x)/π_ref(y_w|x) - log π(y_l|x)/π_ref(y_l|x)))
```

`y_w` preferred response, `y_l` rejected response, `π_ref` reference (SFT) model. DPO simpler, more stable, cheaper - reward model లేదు, RL loop లేదు, hyperparameter-sensitive PPO లేదు. Meta LLaMA 2 మరియు చాలా open-source models DPO లేదా variants use చేస్తున్నాయి.

<ThinkAboutIt>
DPO reward model eliminate చేస్తుంది కానీ inference time లో arbitrary new responses score చేయగల ability కూడా lose అవుతుంది. Explicit reward model (full RLHF pipeline) DPO simplicity కంటే more valuable అయ్యే scenarios ఏవి?
</ThinkAboutIt>

## Constitutional AI: Scale లో Self-Supervision

Anthropic human labellers dependency తగ్గించడానికి **Constitutional AI (CAI)** introduce చేసింది:

1. Principles set (a "constitution") define చేయడం - ఉదా., *"Honest మరియు harmless గా ఉంటూ most helpful response choose చేయి"*
2. Response pairs generate చేసి constitution ని better adhere అయ్యే response judge చేయడానికి **model itself** use చేయడం
3. ఈ AI-generated preferences తో reward model train చేయడం (RLAIF - RL from AI Feedback)

CAI preference collection dramatically scale చేస్తుంది - AI human labelling cost fraction లో millions of comparisons generate మరియు evaluate చేయగలదు. Alignment criteria **explicit మరియు auditable** - constitution లో encode అవుతుంది, labeller biases నుండి implicitly learn అవడం కాదు.

## Red Teaming

ఏ alignment technique perfect కాదు. **Red teaming** model ని deliberately harmful, biased, లేదా untruthful outputs produce చేయించడానికి try చేయడం. Red teamers ఇవి probe చేస్తారు:

- Safety training bypass చేసే jailbreaks
- Demographics అంతటా responses లో systematic biases
- High-stakes domains (medical, legal) లో factual errors
- Refusal boundaries లో inconsistencies

Major labs ఇప్పుడు models release చేయడానికి ముందు extensive red teaming conduct చేస్తున్నాయి, external security researchers, domain experts, మరియు dedicated adversarial testing teams involve అవుతున్నారు.

## Alignment Tax

Alignment కి cost ఉంటుంది. RLHF-trained models కొన్నిసార్లు **overly cautious** అవుతాయి - benign requests refuse చేయడం, unnecessary caveats add చేయడం, perceived risk avoid చేయడానికి bland responses produce చేయడం. ఈ "alignment tax" real engineering challenge:

- Too little alignment → harmful outputs
- Too much alignment → unhelpful, annoying refusals

Alignment research frontier genuine safety boundaries maintain చేస్తూ helpfulness maximise చేసే techniques find చేయడం - superficially "safe" responses pattern-match చేయడం కాదు.
