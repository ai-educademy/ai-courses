# Technical Resume Masterclass - ATS पार करके Interviews तक पहुँचें

आपके resume को human attention के **छह seconds** मिलते हैं - अगर human तक पहुँचे तब। ज़्यादातर large companies में Applicant Tracking System (ATS) आपके resume को parse और rank करता है किसी recruiter के देखने से पहले। ATS software कैसे काम करता है, यह समझना interviews land करने का पहला step है।

## ATS Software असल में कैसे काम करता है

ATS (Applicant Tracking System) एक software है जो resumes को keywords के लिए scan करता है, candidates rank करता है, और job description से match न करने वाली applications filter कर देता है। Workday, Greenhouse, और Lever जैसी companies 95% से ज़्यादा Fortune 500 firms में hiring power करती हैं।

ATS तीन काम करता है:

1. **Parse** करता है resume को structured fields में (name, email, experience, skills)
2. **Match** करता है job description के keywords आपके content से
3. **Rank** करता है आपको बाकी applicants के relative

<FunFact>75% से ज़्यादा resumes ATS software द्वारा reject हो जाते हैं human के पढ़ने से पहले। Well-formatted, keyword-optimised resume आपके odds dramatically बढ़ाता है।</FunFact>

## One-Page Rule

जब तक 15+ years experience न हो, **एक page** में रखें। Google, Amazon, और Meta के recruiters ने public talks में बार-बार confirm किया है। एक focused page हमेशा दो diluted pages से बेहतर है।

## Resume Sections जो Matter करते हैं

### 1. Header
Name, email, phone, location (सिर्फ़ city), LinkedIn URL, GitHub URL, portfolio link। No photo, no date of birth, no marital status।

### 2. Professional Summary (2-3 lines)
एक concise pitch: role, years of experience, key technologies, और एक headline achievement।

> *"Senior Backend Engineer with 6 years building distributed systems in Java and Go. Led migration of payment service to event-driven architecture, reducing latency by 40% and saving £120K annually."*

### 3. Experience (core section)
हर bullet के लिए **XYZ formula** use करें: *Accomplished **X** by doing **Y**, resulting in **Z**.*

| कमज़ोर | Strong (XYZ) |
|------|--------------|
| "Search feature पर काम किया" | "BM25 ranking implement करके search relevance 23% improve की, user engagement 15% बढ़ा" |
| "CI/CD की responsibility थी" | "GitHub Actions में parallel test stages से migrate करके deployment time 45 min से 8 min कम किया" |

### 4. Skills
Technologies honestly list करें। Group करें: **Languages**, **Frameworks**, **Cloud/DevOps**, **Data**। Job description के exact terms match करें - अगर listing "Kubernetes" कहती है तो "K8s" नहीं, "Kubernetes" लिखें।

### 5. Education
Degree, university, graduation year। Relevant coursework या thesis सिर्फ़ तब add करें जब early-career हों।

## Action Verbs जो ATS Filters Pass करें

हर bullet strong verb से शुरू करें: **Designed**, **Implemented**, **Migrated**, **Reduced**, **Orchestrated**, **Optimised**, **Automated**, **Scaled**, **Mentored**, **Delivered**।

कमज़ोर openers avoid करें: "Helped with", "Assisted in", "Was responsible for", "Participated in"।

## हर Job Description के लिए Tailor करना

Job description copy करें। **Top 10 keywords** highlight करें (technologies, methodologies, domain terms)। Ensure करें कि उनमें से कम से कम 7 naturally आपके resume में हों। यह system game करना नहीं - employer की language बोलना है।

एक **master resume** बनाएँ सारे experience के साथ, फिर हर application के लिए tailored copy बनाएँ। Master को single source of truth रखें और per role cut करें।

## Common Mistakes जो Applications Kill करती हैं

- **Columns, tables, या graphics वाले fancy templates** - ATS parse नहीं कर सकता। Single-column, clean layout use करें।
- **Skills के बारे में झूठ** - technical interviews तुरंत expose कर देंगे, और permanently bridges burn होते हैं।
- **हर technology include करना जो कभी touch की** - सिर्फ़ वो list करें जो interview में confidently discuss कर सकें।
- **Images या scanned text वाली PDF** - word processor से text-based PDF export करें।
- **Generic objective statement** - "seeking a challenging role" को targeted professional summary से replace करें जो company और role name करे।
- **Dates omit करना** - interviewers notice करते हैं और assume करते हैं employment gaps hide कर रहे हैं। Transparent रहें; gaps normal हैं।

## GitHub Profile Optimisation

GitHub आपका living portfolio है। Recruiters **check करते हैं**।

- **4-6 best repositories pin करें** clear README files के साथ (problem, solution, tech stack, how to run)
- **Profile README लिखें** (`username/username` repo) short bio और links के साथ
- **Contribution graph maintain करें** - consistency intensity से ज़्यादा matter करती है
- **Old repos clean up करें** - archive करें या embarrassing ones private करें

## Portfolio Links और Personal Sites

Personal site या portfolio हो, तो header में include करें। Well-structured portfolio page में हो:

- **Featured projects** live demos या screenshots के साथ
- **Brief "about me"** section जो resume summary match करे
- **Articles या talks के links**
- Fast-loading और mobile-friendly रखें

Custom domain use करें (e.g., `janedoe.dev`) generic subdomain की बजाय। यह professionalism signal करता है।

## LinkedIn Synergy

LinkedIn और resume **same story** बताएँ different depths के साथ। LinkedIn ज़्यादा detail, media, और recommendations allow करता है। Job titles, dates, और companies exactly match करें - discrepancies red flags raise करती हैं।

Key LinkedIn optimisations:
- **Headline**: Target role और key technologies use करें, सिर्फ़ current title नहीं
- **About section**: Summary expand करें narrative tone और call to action के साथ
- **Featured section**: Best articles, talks, या project demos pin करें
- **Recommendations**: 2-3 managers या senior colleagues से request करें जो specific achievements पर बोल सकें

## Submit करने से पहले Resume Checklist

- [ ] Single-column, no graphics, no tables
- [ ] Contact info complete (email, phone, LinkedIn, GitHub)
- [ ] हर bullet XYZ या action-verb-plus-impact format use करे
- [ ] Skills section job description keywords mirror करे
- [ ] No typos (spell-checker और human reviewer use करें)
- [ ] Text-based PDF के रूप में save
- [ ] One page (unless 15+ years experience)
