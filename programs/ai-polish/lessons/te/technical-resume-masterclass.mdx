# Technical Resume Masterclass — ATS దాటి Interviews లోకి చేరుకోవడం

మీ resume కి **ఆరు సెకన్ల** human attention — అది human వరకు చేరుకుంటే. చాలా large companies లో, Applicant Tracking System (ATS) మీ resume parse చేసి rank చేసి, recruiter చూడడానికి ముందే filter చేస్తుంది. ATS software ఎలా work చేస్తుందో అర్థం చేసుకోవడం interviews land చేయడానికి first step.

## ATS Software Actually ఎలా Work చేస్తుంది

ATS (Applicant Tracking System) resumes ని keywords కోసం scan చేసి, candidates rank చేసి, job description match కాని applications filter out చేసే software. Workday, Greenhouse, Lever వంటి companies Fortune 500 firms లో 95% కంటే ఎక్కువ hiring power చేస్తాయి.

ATS మూడు పనులు చేస్తుంది:

1. Resume ని structured fields గా **parse** చేస్తుంది (name, email, experience, skills)
2. Job description నుండి keywords మీ content తో **match** చేస్తుంది
3. ఇతర applicants తో relative గా మిమ్మల్ని **rank** చేస్తుంది

<FunFact>75% కంటే ఎక్కువ resumes ATS software reject చేసేస్తుంది — human read చేయడానికి ముందే. Well-formatted, keyword-optimised resume మీ odds dramatically increase చేస్తుంది.</FunFact>

## One-Page Rule

15+ years experience లేకపోతే, **one page** కి ఉంచండి. Google, Amazon, Meta recruiters public talks లో repeatedly confirm చేశారు. One focused page two diluted pages ని ఎప్పుడూ beat చేస్తుంది.

## Matter అయ్యే Resume Sections

### 1. Header
Name, email, phone, location (city only), LinkedIn URL, GitHub URL, portfolio link. Photo లేదు, date of birth లేదు, marital status లేదు.

### 2. Professional Summary (2-3 lines)
Concise pitch: role, years of experience, key technologies, headline achievement.

> *"6 years experience ఉన్న Senior Backend Engineer, Java మరియు Go లో distributed systems build చేస్తున్నాను. Payment service ని event-driven architecture కి migration lead చేశాను, latency 40% reduce చేసి annually £120K save చేశాను."*

### 3. Experience (core)
ప్రతి bullet కోసం **XYZ formula** వాడండి: ***Y** చేయడం ద్వారా **X** accomplish చేశాను, **Z** result వచ్చింది.*

| Weak | Strong (XYZ) |
|------|--------------|
| "Search feature మీద work చేశాను" | "BM25 ranking implement చేసి search relevance 23% improve చేశాను, user engagement 15% increase అయింది" |
| "CI/CD responsible" | "GitHub Actions with parallel test stages కి migrate చేసి deployment time 45 min నుండి 8 min కి reduce చేశాను" |

### 4. Skills
Technologies honestly list చేయండి. Group చేయండి: **Languages**, **Frameworks**, **Cloud/DevOps**, **Data**. Job description exact terms match చేయండి — listing "Kubernetes" అంటే "K8s" కాదు "Kubernetes" రాయండి.

### 5. Education
Degree, university, graduation year. Early-career అయితే మాత్రమే relevant coursework లేదా thesis add చేయండి.

## ATS Filters Pass అయ్యే Action Verbs

ప్రతి bullet strong verb తో start చేయండి: **Designed**, **Implemented**, **Migrated**, **Reduced**, **Orchestrated**, **Optimised**, **Automated**, **Scaled**, **Mentored**, **Delivered**.

Weak openers avoid చేయండి: "Helped with", "Assisted in", "Was responsible for", "Participated in".

## ప్రతి Job Description కి Tailor చేయడం

Job description document లోకి copy చేయండి. **Top 10 keywords** highlight చేయండి (technologies, methodologies, domain terms). ఆ 10 లో కనీసం 7 naturally మీ resume లో ఉండేలా చూడండి. ఇది system game చేయడం కాదు — employer language speak చేయడం.

అన్ని experience తో **master resume** create చేయండి, తర్వాత ప్రతి application కోసం copy tailor చేయండి. Master ని single source of truth గా ఉంచి role ప్రకారం cut చేయండి.

## Applications Kill చేసే Common Mistakes

- **Columns, tables, graphics ఉన్న fancy templates** — ATS parse చేయలేదు. Single-column, clean layout వాడండి.
- **Skills గురించి lie చేయడం** — technical interviews immediately expose చేస్తాయి, bridges permanently burn అవుతాయి.
- **Touch చేసిన ప్రతి technology include చేయడం** — interview లో confidently discuss చేయగలిగేవి మాత్రమే list చేయండి.
- **Images లేదా scanned text ఉన్న PDF** — word processor నుండి export చేసిన text-based PDF వాడండి.
- **Generic objective statement** — "seeking a challenging role" బదులు company మరియు role name చేసే targeted professional summary రాయండి.
- **Dates omit చేయడం** — interviewers notice చేస్తారు, employment gaps hide చేస్తున్నారని assume చేస్తారు. Transparent గా ఉండండి; gaps normal.

## GitHub Profile Optimisation

మీ GitHub living portfolio. Recruiters **check చేస్తారు**.

- **Best 4-6 repositories pin** చేయండి — clear README files తో (problem, solution, tech stack, how to run)
- **Profile README write** చేయండి (`username/username` repo) — short bio మరియు links తో
- **Contribution graph maintain** చేయండి — consistency intensity కంటే ఎక్కువ matter
- **Old repos clean up** చేయండి — embarrassing ఏదైనా archive లేదా private చేయండి

## Portfolio Links మరియు Personal Sites

Personal site లేదా portfolio ఉంటే header లో include చేయండి. Well-structured portfolio page లో:

- Live demos లేదా screenshots తో **featured projects**
- Resume summary match అయ్యే brief **"about me"** section
- మీరు ఇచ్చిన **articles లేదా talks** links
- Fast-loading మరియు mobile-friendly ఉంచండి

Custom domain (e.g., `janedoe.dev`) generic subdomain కంటే better. Professionalism మరియు attention to detail signal చేస్తుంది.

## LinkedIn Synergy

LinkedIn మరియు resume **same story** చెప్పాలి different depths లో. LinkedIn ఎక్కువ detail, media, recommendations allow చేస్తుంది. Job titles, dates, companies exactly match అవ్వాలి — discrepancies red flags raise చేస్తాయి.

Key LinkedIn optimisations:
- **Headline**: Current title మాత్రమే కాదు, target role మరియు key technologies వాడండి
- **About section**: Narrative tone మరియు call to action తో summary expand చేయండి
- **Featured section**: Best articles, talks, లేదా project demos pin చేయండి
- **Recommendations**: Specific achievements speak చేయగల managers లేదా senior colleagues నుండి 2-3 request చేయండి

## Submit చేయడానికి ముందు Resume Checklist

- [ ] Single-column, graphics లేదు, tables లేదు
- [ ] Contact info complete (email, phone, LinkedIn, GitHub)
- [ ] ప్రతి bullet XYZ లేదా action-verb-plus-impact format వాడుతోంది
- [ ] Skills section job description keywords mirror చేస్తోంది
- [ ] Typos లేవు (spell-checker మరియు human reviewer వాడండి)
- [ ] Text-based PDF గా save చేయబడింది
- [ ] One page (15+ years experience అయితే తప్ప)
