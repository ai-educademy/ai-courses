## Qu'est-ce qui rend une pile monotone ?

Une **pile monotone** (monotonic stack) maintient les éléments dans un ordre strictement croissant ou décroissant du bas vers le haut. Avant de push un nouvel élément, on pop tout ce qui viole l'invariant. Cette seule idée permet de résoudre en O(n) des problèmes qui semblent exiger O(n²).

```
Decreasing stack processing [2, 1, 5, 3, 4]:

Push 2 → [2]
Push 1 → [2, 1]         (1 < 2, invariant holds)
Pop 1, Pop 2, Push 5 → [5]  (5 > both)
Push 3 → [5, 3]
Pop 3, Push 4 → [5, 4]  (4 > 3)
```

## Next Greater Element

Pour chaque élément, trouver le premier élément plus grand à sa droite. L'approche brute force parcourt vers la droite pour chaque position - O(n²). Une pile décroissante le fait en un seul passage.

```python
def next_greater(nums):
    result = [-1] * len(nums)
    stack = []
    for i, num in enumerate(nums):
        while stack and nums[stack[-1]] < num:
            result[stack.pop()] = num
        stack.append(i)
    return result
```

Quand on pop l'index `j`, l'élément courant est son next greater - c'est garanti car tout ce qui se trouvait entre les deux était plus petit (ils avaient été pop avant ou n'ont jamais bloqué `j`).

## Largest Rectangle in Histogram

Le **Largest Rectangle in Histogram** de LeetCode est le joyau des problèmes de pile monotone.

**Intuition clé :** pour chaque barre, trouver jusqu'où elle peut s'étendre à gauche et à droite en tant que barre la plus courte. Utilisez une pile croissante - quand une barre est pop, la barre qui la pop définit sa frontière droite, et le nouveau sommet de la pile définit sa frontière gauche.

```python
def largest_rectangle(heights):
    stack, max_area = [], 0
    for i, h in enumerate(heights + [0]):
        while stack and heights[stack[-1]] > h:
            height = heights[stack.pop()]
            width = i if not stack else i - stack[-1] - 1
            max_area = max(max_area, height * width)
        stack.append(i)
    return max_area
```

La sentinelle `[0]` à la fin force le traitement de toutes les barres restantes.

## Trapping Rain Water - Approche par pile

Au lieu de la méthode classique à deux pointeurs, une pile décroissante calcule l'eau piégée couche par couche. Quand une barre plus haute apparaît, on pop les barres plus courtes et on calcule l'eau piégée entre la barre pop, la barre courante et le nouveau sommet de la pile.

```
Heights:  [3, 1, 2, 1, 3]
Water:       [2, 1, 2]  → total = 5
```

<FunFact>Trapping Rain Water possède au moins quatre solutions O(n) distinctes - deux pointeurs, pile monotone, tableaux de préfixes max et une approche optimisée en un seul passage. Les intervieweurs adorent voir plusieurs stratégies.</FunFact>

## Monotonic Queue - Sliding Window Maximum

Un **deque monotone** (file à double extrémité) maintient un ordre décroissant et permet des suppressions efficaces aux deux extrémités. Pour le **Sliding Window Maximum** (LeetCode 239) :

1. Avant d'ajouter l'index `i`, pop tous les éléments arrière plus petits que `nums[i]`
2. Push `i` à l'arrière
3. L'avant est toujours le maximum de la fenêtre courante
4. Retirer l'avant s'il est sorti de la fenêtre

```python
from collections import deque

def max_sliding_window(nums, k):
    dq, result = deque(), []
    for i, num in enumerate(nums):
        while dq and nums[dq[-1]] <= num:
            dq.pop()
        dq.append(i)
        if dq[0] <= i - k:
            dq.popleft()
        if i >= k - 1:
            result.append(nums[dq[0]])
    return result
```

Chaque élément entre et sort du deque au plus une fois - **O(n)** au total.

## Le pattern - Maintenir l'invariant pendant le traitement

Chaque problème de pile/file monotone suit un même modèle :

1. **Définir l'invariant** (croissant ou décroissant)
2. **Pop les éléments qui violent l'invariant** - et extraire l'information utile de chaque pop
3. **Push l'élément courant**
4. L'*événement de pop* est le moment où la réponse est calculée
