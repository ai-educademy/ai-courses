## Représentation des intervalles

Un interval est une paire `[start, end]`. La plupart des problèmes d'intervalles commencent par une étape cruciale : **trier par temps de début**. Cela transforme une liste chaotique en une séquence que l'on peut parcourir de gauche à droite avec de simples comparaisons.

```
Unsorted: [3,6] [1,4] [8,10] [2,5]
Sorted:   [1,4] [2,5] [3,6] [8,10]
```

## Merge des intervalles qui se chevauchent

Le problème fondamental d'intervalles (LeetCode 56). Après le tri, deux intervalles consécutifs se chevauchent si le début courant ≤ la fin du précédent.

```python
def merge(intervals):
    intervals.sort()
    merged = [intervals[0]]
    for start, end in intervals[1:]:
        if start <= merged[-1][1]:
            merged[-1][1] = max(merged[-1][1], end)
        else:
            merged.append([start, end])
    return merged
```

Parcourons `[1,4] [2,5] [3,6] [8,10]` :
- `[2,5]` chevauche `[1,4]` → merge en `[1,5]`
- `[3,6]` chevauche `[1,5]` → merge en `[1,6]`
- `[8,10]` pas de chevauchement → nouvel interval

Résultat : `[1,6] [8,10]`

## Insert Interval

Étant donné des intervalles triés et non chevauchants ainsi qu'un nouvel interval, insérez-le et fusionnez si nécessaire (LeetCode 57). Divisez en trois phases :

1. Ajoutez tous les intervalles qui se terminent avant le début du nouveau
2. Fusionnez tous les intervalles chevauchants avec le nouveau
3. Ajoutez les intervalles restants sans modification

<ThinkAboutIt>Insert Interval est souvent présenté sans que l'étape de merge soit évidente. Comment reconnaîtriez-vous qu'il s'agit d'un problème de merge lors d'un entretien ?</ThinkAboutIt>

## Meeting Rooms I — Peut-on tout assister ?

Triez par temps de début. Si le début d'un interval est avant la fin du précédent, il y a un conflit.

```python
def can_attend(intervals):
    intervals.sort()
    for i in range(1, len(intervals)):
        if intervals[i][0] < intervals[i - 1][1]:
            return False
    return True
```

## Meeting Rooms II — Nombre minimum de salles

C'est ici que l'**approche par événements** brille. Séparez les débuts et fins en deux listes triées, puis balayez :

```python
import heapq

def min_rooms(intervals):
    intervals.sort()
    heap = []
    for start, end in intervals:
        if heap and heap[0] <= start:
            heapq.heappop(heap)
        heapq.heappush(heap, end)
    return len(heap)
```

Le heap suit la réunion se terminant le plus tôt. Si elle se termine avant le début de la suivante, on réutilise la salle. La taille maximale du heap est la réponse.

## Non-Overlapping Intervals — Minimum de suppressions

Étant donné des intervalles, trouvez le nombre minimum à supprimer pour qu'aucun ne se chevauche (LeetCode 435). Greedy : triez par **temps de fin**, gardez toujours l'interval qui se termine le plus tôt — cela laisse le plus de place pour les futurs intervalles.

```python
def erase_overlap(intervals):
    intervals.sort(key=lambda x: x[1])
    count, prev_end = 0, float('-inf')
    for start, end in intervals:
        if start >= prev_end:
            prev_end = end
        else:
            count += 1
    return count
```

## Intersection de listes d'intervalles

Étant donné deux listes d'intervalles triées, trouvez toutes les intersections (LeetCode 986). Utilisez deux pointeurs — un par liste. L'intersection de `[a1, a2]` et `[b1, b2]` est `[max(a1, b1), min(a2, b2)]` si elle est valide (début ≤ fin). Avancez le pointeur dont l'interval se termine en premier.

## La technique du Sweep Line

Au lieu de comparer les intervalles deux à deux, décomposez chacun en événements :

```
[1,4] → +1 at 1, -1 at 4
[2,6] → +1 at 2, -1 at 6
```

Triez les événements par temps. Balayez de gauche à droite en maintenant un compteur courant. Le compteur maximum répond aux questions de « chevauchement maximal ». Cela généralise Meeting Rooms II à des problèmes arbitraires de traitement d'événements.

## Calendar Booking

**My Calendar I** (LeetCode 729) : maintenez les intervalles réservés et rejetez les chevauchements. **My Calendar II** : autorisez les chevauchements simples mais rejetez les triple-bookings. Le sweep line rend la détection de chevauchements multi-couches simple et directe.

Cherchez ces indices dans les énoncés de problèmes :

- Des mots comme « overlapping », « conflicting », « schedule », « range »
- Des paires de valeurs début/fin
- Des questions sur « combien de simultanés » ou « minimum de ressources »
- Des données représentables comme des segments sur une droite numérique
