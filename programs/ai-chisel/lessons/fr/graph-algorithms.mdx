## Les graphes modélisent le monde connecté

Amitiés, réseaux routiers, hyperliens, voies neuronales - chaque fois que des entités sont reliées par des relations, vous avez un **graph**. Formellement, un graph G = (V, E) se compose de sommets (nodes) et d'arêtes (edges).

Les graphes peuvent être **dirigés** (rues à sens unique) ou **non dirigés** (bidirectionnels), et les arêtes peuvent porter des **poids** (distances, coûts, latences).

## Représentation des graphes

Deux représentations courantes :

| Représentation | Espace | Recherche d'arête | Idéal pour |
|---------------|--------|-------------------|------------|
| **Liste d'adjacence** | O(V + E) | O(degré) | Graphes creux (la plupart des réseaux réels) |
| **Matrice d'adjacence** | O(V²) | O(1) | Graphes denses, vérification rapide d'existence d'arête |

```
// Adjacency list example
graph = {
    A: [(B, 4), (C, 1)],
    B: [(D, 1)],
    C: [(B, 2), (D, 5)],
    D: []
}
```

## Breadth-First Search (BFS)

Le BFS explore **niveau par niveau**, visitant tous les voisins avant d'aller plus en profondeur. Il utilise une **file d'attente** (queue).

```
function bfs(graph, start):
    visited = {start}
    queue = [start]
    while queue is not empty:
        node = queue.dequeue()
        process(node)
        for neighbour in graph[node]:
            if neighbour not in visited:
                visited.add(neighbour)
                queue.enqueue(neighbour)
```

**Propriété clé :** Le BFS trouve le **plus court chemin** dans les graphes non pondérés car il traite les nœuds dans l'ordre de distance depuis la source.

### Cas d'usage IA du BFS

- **Suggestions d'amis sur les réseaux sociaux** - trouver des personnes à 2–3 connexions.
- **Robots d'indexation web** - indexer les pages couche par couche depuis une URL de départ.
- **Résolution de puzzles** - trouver le nombre minimum de mouvements (ex. états du Rubik's cube).

## Depth-First Search (DFS)

Le DFS va **aussi profondément que possible** avant de revenir en arrière. Il utilise une **pile** (stack) ou la récursion.

```
function dfs(graph, node, visited):
    visited.add(node)
    process(node)
    for neighbour in graph[node]:
        if neighbour not in visited:
            dfs(graph, neighbour, visited)
```

### Applications du DFS

- **Tri topologique** - ordonner les tâches pour que chaque dépendance vienne en premier (systèmes de build, prérequis de cours).
- **Détection de cycles** - si le DFS revisité un nœud actuellement sur la pile d'appels, un cycle existe.
- **Résolution de labyrinthes** - explorer un chemin entièrement avant d'essayer les alternatives.
- **Composantes connexes** - lancer un DFS depuis chaque nœud non visité pour identifier des clusters séparés.

## Algorithme de Dijkstra - Plus court chemin avec poids

Lorsque les arêtes ont des coûts différents, le BFS ne suffit plus. L'algorithme de Dijkstra trouve le plus court chemin pondéré d'une source vers tous les autres nœuds.

**Idée centrale :** Toujours étendre le **nœud non visité le moins coûteux** à l'aide d'un min-heap (priority queue).

```
function dijkstra(graph, source):
    dist = {source: 0}          // all others: infinity
    pq = MinHeap([(0, source)])
    while pq is not empty:
        (cost, node) = pq.extractMin()
        if cost > dist[node]: continue
        for (neighbour, weight) in graph[node]:
            newDist = cost + weight
            if newDist < dist.getOrDefault(neighbour, INF):
                dist[neighbour] = newDist
                pq.insert((newDist, neighbour))
    return dist
```

**Complexité temporelle :** O((V + E) log V) avec un tas binaire.

<FunFact>Google Maps n'exécute pas un simple Dijkstra sur l'ensemble du réseau routier. Il utilise des hiérarchies de contraction et la recherche A* - mais les deux reposent sur l'intuition fondamentale de Dijkstra : l'expansion gloutonne par distance connue la plus courte.</FunFact>

### Cas d'usage IA de Dijkstra

- **Planification d'itinéraires** - les applications de navigation trouvant le trajet le plus rapide.
- **Routage réseau** - les paquets internet choisissant le chemin à plus faible latence.
- **Graphes de connaissances** - trouver la connexion sémantique la plus courte entre des concepts.

## Détection de cycles et composantes connexes

### Détection de cycles

Dans un **graphe dirigé**, suivez les nœuds dans trois états : non visité, en cours (sur le chemin DFS actuel) et terminé. Rencontrer un nœud en cours signifie qu'un cycle existe.

Dans un **graphe non dirigé**, un cycle existe si le DFS atteint un nœud visité qui n'est pas le parent immédiat.

### Composantes connexes

Lancez un BFS ou DFS depuis chaque nœud non visité. Chaque exécution découvre une composante connexe. Dans les réseaux sociaux, les composantes révèlent des communautés isolées.

<ThinkAboutIt>Un système de détection de fraude par IA modélise les transactions sous forme de graphe où les comptes sont des nœuds et les transferts sont des arêtes. Comment les composantes connexes pourraient-elles aider à identifier des réseaux de blanchiment d'argent ?</ThinkAboutIt>

## Points clés à retenir

1. **BFS** explore niveau par niveau - idéal pour les plus courts chemins dans les graphes non pondérés et la découverte basée sur le voisinage.
2. **DFS** plonge en profondeur - la colonne vertébrale du tri topologique, de la détection de cycles et de l'analyse des composantes.
3. **Dijkstra** étend le BFS aux graphes pondérés grâce à une priority queue - le fondement de la planification d'itinéraires moderne.
4. Les algorithmes de graphes sont le moteur derrière les cartes, les réseaux sociaux, les gestionnaires de paquets et les graphes de connaissances.

<FunFact>Le World Wide Web contient plus de 50 milliards de pages indexées. Les robots d'indexation des moteurs de recherche utilisent des variantes du BFS pour découvrir et ré-indexer les pages, en priorisant selon l'importance de la page (PageRank).</FunFact>
