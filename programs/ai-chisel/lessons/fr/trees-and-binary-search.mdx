## Les arbres sont partout en IA

Chaque fois qu'un filtre anti-spam classe un e-mail ou qu'une base de données récupère votre commande en millisecondes, une structure de données arborescente fait le gros du travail. Les arbres modélisent des **décisions hiérarchiques** — chaque nœud réduit l'espace de recherche jusqu'à atteindre une réponse.

## Arbres binaires de recherche (BST)

Un BST stocke les éléments de sorte que pour tout nœud, toutes les valeurs de son sous-arbre **gauche** sont plus petites et toutes celles de son sous-arbre **droit** sont plus grandes.

### Recherche

```
function search(node, target):
    if node is null: return null
    if target == node.value: return node
    if target < node.value: return search(node.left, target)
    return search(node.right, target)
```

Chaque comparaison élimine environ la moitié des nœuds restants — O(log n) sur un arbre équilibré.

### Insertion

```
function insert(node, value):
    if node is null: return new Node(value)
    if value < node.value:
        node.left = insert(node.left, value)
    else:
        node.right = insert(node.right, value)
    return node
```

### Suppression

La suppression est l'opération la plus délicate. Trois cas se présentent :

1. **Nœud feuille** — il suffit de le supprimer.
2. **Un seul enfant** — remplacer le nœud par son enfant.
3. **Deux enfants** — échanger la valeur du nœud avec son successeur in-order (la plus petite valeur du sous-arbre droit), puis supprimer ce successeur.

<Quiz question="Quel est le successeur in-order d'un nœud dans un BST ?" options="Le parent du nœud|Le plus grand nœud du sous-arbre gauche|Le plus petit nœud du sous-arbre droit|La racine de l'arbre" answer="2" explanation="Le successeur in-order est la plus petite valeur du sous-arbre droit — on va à droite une fois, puis à gauche aussi loin que possible." />

## Équilibré vs non équilibré

Si vous insérez des données triées dans un BST simple, il dégénère en liste chaînée — O(n) par opération. Les arbres **auto-équilibrants** corrigent ce problème :

| Arbre | Règle d'équilibre | Utilisation typique |
|-------|-------------------|---------------------|
| AVL | Différence de hauteur ≤ 1 entre sous-arbres | Recherches en mémoire |
| Red-Black | Règles de rotation basées sur la couleur | Bibliothèques standard (ex. Java TreeMap) |
| B-Tree | Plusieurs clés par nœud, toutes les feuilles à la même profondeur | Bases de données et systèmes de fichiers |

<FunFact>PostgreSQL et MySQL utilisent tous deux des variantes de B-tree comme structure d'index par défaut. Un B-tree avec un facteur de branchement de 100 peut indexer un milliard de lignes en seulement cinq niveaux.</FunFact>

## Parcours d'arbres

Les parcours visitent chaque nœud exactement une fois. Les trois ordres classiques ne diffèrent que par le **moment** où l'on traite le nœud courant par rapport à ses enfants.

| Parcours | Ordre | Cas d'utilisation |
|----------|-------|-------------------|
| **In-order** | Gauche → Nœud → Droite | Produit une sortie triée à partir d'un BST |
| **Pre-order** | Nœud → Gauche → Droite | Sérialiser / cloner un arbre |
| **Post-order** | Gauche → Droite → Nœud | Supprimer un arbre en toute sécurité ; évaluer des arbres d'expressions |

```
function inOrder(node):
    if node is null: return
    inOrder(node.left)
    process(node)
    inOrder(node.right)
```

## Tas et files de priorité

Un **tas** (heap) est un arbre binaire complet où chaque parent est plus grand (max-heap) ou plus petit (min-heap) que ses enfants. Il permet :

- **Insertion** — ajouter à la fin, remonter (bubble up). O(log n).
- **Extraction min/max** — retirer la racine, déplacer le dernier élément à la racine, descendre (sift down). O(log n).

Les files de priorité construites sur des tas alimentent :
- **L'algorithme de Dijkstra** (prochain cours !) — toujours étendre le nœud non visité le plus proche.
- **L'ordonnancement des tâches** — les ordonnanceurs du système choisissent le processus de plus haute priorité.
- **Le beam search en NLP** — conserver les top-k complétions de phrases les plus prometteuses.

```
function siftUp(heap, index):
    while index > 0:
        parent = (index - 1) / 2
        if heap[index] < heap[parent]:
            swap(heap, index, parent)
            index = parent
        else: break
```

## Applications réelles en IA

- Les **arbres de décision** en machine learning divisent les données à chaque nœud en utilisant la caractéristique qui sépare le mieux les classes — une application directe de la structure arborescente.
- Les **arbres syntaxiques** en NLP représentent la structure grammaticale, permettant des tâches comme l'analyse de sentiments et la traduction automatique.
- Le **Monte Carlo Tree Search (MCTS)** alimente les IA joueuses comme AlphaGo, explorant les coups possibles sous forme d'arbre d'états de jeu.
- Les **arbres de Huffman** compressent les données en attribuant des codes plus courts aux symboles fréquents — utilisés dans la compression des poids de modèles.

<FunFact>Le classifieur par arbre de décision de scikit-learn construit un arbre où chaque nœud interne teste un seuil de caractéristique. Un modèle entraîné de profondeur 20 peut représenter plus d'un million de chemins de décision distincts.</FunFact>

## Points clés

1. Les BST offrent une recherche, insertion et suppression en O(log n) — mais uniquement lorsqu'ils sont équilibrés.
2. L'ordre de parcours détermine quelles informations vous extrayez d'un arbre.
3. Les tas offrent un accès en O(1) à la valeur extrême et une insertion en O(log n) — idéal pour les algorithmes d'IA basés sur les priorités.
4. Les arbres sont au cœur de la prise de décision en ML, bases de données, compilateurs et IA de jeux.
