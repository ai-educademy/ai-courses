## Stack को Monotonic क्या बनाता है?

एक **monotonic stack** elements को bottom से top तक strictly increasing या decreasing order में maintain करता है। नया element push करने से पहले, आप उन सभी elements को pop कर देते हैं जो invariant को violate करते हैं। यह एक simple idea O(n²) लगने वाली problems को O(n) में solve कर देता है।

```
Decreasing stack processing [2, 1, 5, 3, 4]:

Push 2 → [2]
Push 1 → [2, 1]         (1 < 2, invariant holds)
Pop 1, Pop 2, Push 5 → [5]  (5 > both)
Push 3 → [5, 3]
Pop 3, Push 4 → [5, 4]  (4 > 3)
```

## Next Greater Element

हर element के लिए, उसके दाईं ओर पहला बड़ा element ढूंढना है। Brute force हर position से right की तरफ scan करता है — O(n²)। एक monotonic decreasing stack यह काम एक ही pass में कर देता है।

```python
def next_greater(nums):
    result = [-1] * len(nums)
    stack = []
    for i, num in enumerate(nums):
        while stack and nums[stack[-1]] < num:
            result[stack.pop()] = num
        stack.append(i)
    return result
```

जब हम index `j` को pop करते हैं, तो current element उसका next greater है — यह guarantee है क्योंकि बीच के सारे elements छोटे थे (वे पहले pop हो गए थे या `j` को कभी block नहीं किया)।

## Stock Span Problem

*हर दिन के लिए, कितने लगातार दिनों (आज सहित) की price आज की price से ≤ थी?* एक monotonic decreasing stack use करें जिसमें `(price, index)` store हो। जब आप pop करते हैं, current index और नए stack top के बीच का gap span देता है।

## Largest Rectangle in Histogram

LeetCode का **Largest Rectangle in Histogram** monotonic stack problems का सबसे शानदार problem है।

**Key insight:** हर bar के लिए, पता करें कि वह सबसे छोटी bar होते हुए कितना left और right extend कर सकती है। एक increasing stack use करें — जब कोई bar pop होती है, popping bar उसकी right boundary define करती है, और नया stack top left boundary define करता है।

```python
def largest_rectangle(heights):
    stack, max_area = [], 0
    for i, h in enumerate(heights + [0]):
        while stack and heights[stack[-1]] > h:
            height = heights[stack.pop()]
            width = i if not stack else i - stack[-1] - 1
            max_area = max(max_area, height * width)
        stack.append(i)
    return max_area
```

अंत में sentinel `[0]` बाकी सभी bars को process करने के लिए force करता है।

## Trapping Rain Water — Stack Approach

Classic two-pointer method की जगह, एक monotonic decreasing stack trapped water को layer by layer calculate करता है। जब एक ऊँची bar आती है, छोटी bars को pop करें और popped bar, current bar, और नए stack top के बीच trapped water calculate करें।

```
Heights:  [3, 1, 2, 1, 3]
Water:       [2, 1, 2]  → total = 5
```

<FunFact>Trapping Rain Water के कम से कम चार अलग-अलग O(n) solutions हैं — two-pointer, monotonic stack, prefix max arrays, और एक single-pass optimised approach। Interviewers को multiple strategies देखना बहुत पसंद है।</FunFact>

## Monotonic Queue — Sliding Window Maximum

एक **monotonic deque** (double-ended queue) decreasing order maintain करता है और दोनों ends से efficiently remove कर सकता है। **Sliding Window Maximum** (LeetCode 239) के लिए:

1. Index `i` add करने से पहले, back के सभी elements जो `nums[i]` से छोटे हैं उन्हें pop करें
2. `i` को back में push करें
3. Front हमेशा current window का maximum होता है
4. अगर front window के बाहर चला गया है तो उसे remove करें

```python
from collections import deque

def max_sliding_window(nums, k):
    dq, result = deque(), []
    for i, num in enumerate(nums):
        while dq and nums[dq[-1]] <= num:
            dq.pop()
        dq.append(i)
        if dq[0] <= i - k:
            dq.popleft()
        if i >= k - 1:
            result.append(nums[dq[0]])
    return result
```

हर element deque में ज़्यादा से ज़्यादा एक बार enter और leave करता है — कुल **O(n)**।

## Pattern — Process करते समय Invariant Maintain करें

हर monotonic stack/queue problem एक ही template follow करती है:

1. **Invariant define करें** (increasing या decreasing)
2. **Violators को pop करें** — और हर pop से useful information निकालें
3. **Current element push करें**
4. *Pop event* वह जगह है जहाँ answer compute होता है
