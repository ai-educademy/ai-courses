## Trie क्या है?

**Trie** (उच्चारण "try") एक tree जैसी structure है जहाँ हर node एक single character represent करता है। Root से marked nodes तक के paths stored words spell करते हैं। Hash maps के विपरीत, tries आपको *एक prefix share करने वाले सभी words* query करने देते हैं बिना हर key scan किए।

```
        root
       / | \
      a   b   c
     /     \
    p       a
   / \       \
  p   e      t
  |
  l
  e
```

हर path एक word encode करता है: **apple**, **ape**, **bat**, **cat**।

## TrieNode Class

हर node में children का एक map और word completion indicate करने वाला flag होता है।

```python
class TrieNode:
    def __init__(self):
        self.children = {}
        self.is_end = False
```

एक पूरा Trie इसे insert, search, और prefix operations के साथ wrap करता है।

## Core Operations

### Insert - O(m) जहाँ m = word की length

Character by character चलें, ज़रूरत पड़ने पर nodes बनाएं, फिर last node को mark करें।

```python
def insert(self, word):
    node = self.root
    for ch in word:
        if ch not in node.children:
            node.children[ch] = TrieNode()
        node = node.children[ch]
    node.is_end = True
```

### Search और startsWith

**Search** path follow करता है और `is_end` check करता है। **startsWith** सिर्फ़ check करता है कि path exist करता है या नहीं - इसे `is_end = True` की ज़रूरत नहीं।

## DFS के साथ Autocomplete

किसी prefix के लिए suggestions देने के लिए, prefix node तक navigate करें, फिर DFS run करें और नीचे के सभी words collect करें।

```python
def autocomplete(self, prefix):
    node = self._find(prefix)
    if not node:
        return []
    results = []
    self._dfs(node, prefix, results)
    return results
```

`_dfs` helper children में recurse करता है और जब भी `is_end` `True` होता है तो results में append करता है।

## Wildcard Search - Word Dictionary

LeetCode की **Design Add and Search Words Data Structure** problem में `.` को wildcard के रूप में handle करना होता है जो किसी भी single character से match करता है।

Search के दौरान, जब `.` मिले तो *हर* child में branch करें और recursively continue करें। यह एक simple traversal को bounded backtracking search में बदल देता है।

```python
def search_wild(self, node, word, i):
    if i == len(word):
        return node.is_end
    if word[i] == '.':
        return any(
            self.search_wild(c, word, i + 1)
            for c in node.children.values()
        )
    if word[i] not in node.children:
        return False
    return self.search_wild(
        node.children[word[i]], word, i + 1
    )
```

## Longest Common Prefix

Root से trie में चलें। Longest common prefix वहाँ खत्म होता है जहाँ पहला node एक से ज़्यादा children रखता है या `is_end` flag मिलता है।

## Tries के साथ IP Routing

Routers IP prefixes को एक **binary trie** में store करते हैं - हर bit (0 या 1) एक branch बनाता है। Longest prefix match सबसे specific route ढूँढता है। इसी तरह आपके packets internet पर अपना रास्ता खोजते हैं।

<FunFact>Internet backbone routers हर second अरबों longest-prefix-match lookups करते हैं, hardware-optimised tries की मदद से जो TCAMs नाम के specialised chips में बने होते हैं।</FunFact>

## Compressed Tries (Radix Trees)

Standard tries single-child chains पर space waste करते हैं। एक **radix tree** इन chains को merge करके single edges बनाता है जिनमें multi-character labels होते हैं।

```
Standard:  r → u → n → n → i → n → g
Radix:     r → "unning"
```

इससे node count काफ़ी कम हो जाता है और इसे Linux kernel routing tables और HTTP routers जैसे Go के `httprouter` में use किया जाता है।

## Time और Space Complexity

| Operation    | Trie   | Hash Map |
|--------------|--------|----------|
| Insert       | O(m)   | O(m)     |
| Search       | O(m)   | O(m)     |
| Prefix query | O(m+k) | O(n·m)   |

यहाँ *m* word की length है, *k* results की count है, और *n* total keys हैं। Prefix-heavy workloads में tries का दबदबा है।

- Strings पर wildcard या regex-like matching
- एक set में longest common prefix
- IP routing / longest prefix match
- Edit-distance variations के साथ spell checking
