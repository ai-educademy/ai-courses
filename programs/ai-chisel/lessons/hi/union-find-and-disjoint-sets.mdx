## Connectivity की समस्या

मान लीजिए आपके पास nodes का एक network है, और बार-बार दो तरह की queries आ रही हैं: *node A को node B से connect करो* और *क्या A और B connected हैं?* अगर हर query के लिए BFS/DFS चलाएँ तो पूरा graph traverse करना पड़ेगा - लाखों operations के लिए ये बहुत slow है।

**Union-Find** दोनों queries को लगभग constant time में handle करता है।

## Naive Approach - Quick Find

हर node के लिए एक component ID store करो। Connectivity check करने के लिए IDs compare करो - O(1)। लेकिन दो components merge करने के लिए एक पूरे group की हर ID update करनी पड़ेगी - O(n) per union।

```
IDs:  [0, 0, 0, 3, 3]
       └─group 0─┘ └group 3┘
union(0, 3) → update all 3s to 0 → O(n)
```

हमें कुछ बेहतर चाहिए।

## Union-Find Data Structure

हर node एक **parent** की तरफ point करता है। हर tree का root उस component का representative होता है। दो nodes तभी connected हैं जब उनका root same हो।

```python
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n
```

## Find with Path Compression

Deep tree में parent pointers follow करना slow हो सकता है। **Path compression** tree को flat कर देता है - हर visited node को directly root से जोड़ देता है।

```python
def find(self, x):
    if self.parent[x] != x:
        self.parent[x] = self.find(self.parent[x])
    return self.parent[x]
```

Node 4 को find करने से पहले और बाद में:

```
Before:  0←1←2←3←4     After:  0←1
                                0←2
                                0←3
                                0←4
```

## Union by Rank

हमेशा छोटे tree को बड़े tree के नीचे attach करो। इससे tree की height logarithmic रहती है और degenerate chains नहीं बनतीं।

```python
def union(self, x, y):
    rx, ry = self.find(x), self.find(y)
    if rx == ry:
        return False
    if self.rank[rx] < self.rank[ry]:
        rx, ry = ry, rx
    self.parent[ry] = rx
    if self.rank[rx] == self.rank[ry]:
        self.rank[rx] += 1
    return True
```

## Kruskal's MST Algorithm

Union-Find **Kruskal's algorithm** को power करता है - Minimum Spanning Trees के लिए:

1. सभी edges को weight के हिसाब से sort करो
2. हर edge (u, v) के लिए, अगर `find(u) ≠ find(v)` है, तो उसे add करो और `union(u, v)` करो
3. जब n − 1 edges हो जाएँ तो रुक जाओ

Union-Find cycles prevent करता है - अगर दोनों endpoints का root same है तो edge reject हो जाती है। Total time: **O(E log E)** - sorting dominate करती है।

<ThinkAboutIt>Kruskal's algorithm cycles detect करने के लिए Union-Find use करता है। DFS से cycle detect कैसे करोगे, और यहाँ Union-Find ज़्यादा natural क्यों है?</ThinkAboutIt>

## Classic Problems

### Number of Connected Components

Union-Find को n nodes से initialise करो। हर edge को `union` से process करो। अंत में distinct roots count करो।

### Accounts Merge (LeetCode 721)

हर account में emails हैं। अगर दो accounts में कोई common email है, तो वो same person के हैं। Shared emails के basis पर accounts को union करो, फिर group और sort करो।

### Redundant Connection (LeetCode 684)

एक tree दिया है जिसमें एक extra edge है - redundant edge ढूँढो। Edges को order में process करो - जिस पहली edge पर `union` `False` return करे (already connected), वही answer है।

<FunFact>Union-Find दोनों optimisations के साथ O(α(n)) में चलता है - इसका proof Tarjan ने 1975 में publish किया था और इसे computer science में सबसे elegant amortised analyses में से एक माना जाता है।</FunFact>

## Union-Find vs BFS/DFS - कब क्या चुनें

| Scenario | सबसे अच्छा choice |
|----------|-------------------|
| बार-बार connectivity queries | Union-Find |
| Single traversal / shortest path | BFS/DFS |
| Incremental edge additions | Union-Find |
| Edge removals (dynamic graph) | BFS/DFS (या offline UF) |
| Kruskal's MST | Union-Find |
| Topological sort / directed graph में cycle | DFS |

Union-Find तब shine करता है जब आप edges *incrementally* process कर रहे हों और fast connectivity checks चाहिए। अगर सिर्फ एक बार traverse करना है या path information चाहिए, तो BFS/DFS use करें।
