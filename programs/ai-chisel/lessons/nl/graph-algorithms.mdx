## Graphs modelleren de verbonden wereld

Vriendschappen, wegennetwerken, hyperlinks, neurale paden — wanneer entiteiten door relaties verbonden zijn, heb je een **graph**. Formeel bestaat een graph G = (V, E) uit vertices (nodes) en edges (verbindingen).

Graphs kunnen **directed** (eenrichtingsverkeer) of **undirected** (tweerichtingsverkeer) zijn, en edges kunnen **weights** (afstanden, kosten, latenties) bevatten.

## Graphs representeren

Twee veelgebruikte representaties:

| Representatie | Ruimte | Edge-lookup | Geschikt voor |
|--------------|--------|-------------|---------------|
| **Adjacency list** | O(V + E) | O(graad) | Sparse graphs (de meeste real-world netwerken) |
| **Adjacency matrix** | O(V²) | O(1) | Dense graphs, snelle edge-existentiecontroles |

```
// Adjacency list example
graph = {
    A: [(B, 4), (C, 1)],
    B: [(D, 1)],
    C: [(B, 2), (D, 5)],
    D: []
}
```

## Breadth-First Search (BFS)

BFS verkent **niveau voor niveau** en bezoekt alle buren voordat het dieper gaat. Het gebruikt een **queue**.

```
function bfs(graph, start):
    visited = {start}
    queue = [start]
    while queue is not empty:
        node = queue.dequeue()
        process(node)
        for neighbour in graph[node]:
            if neighbour not in visited:
                visited.add(neighbour)
                queue.enqueue(neighbour)
```

**Belangrijke eigenschap:** BFS vindt het **kortste pad** in ongewogen graphs omdat het nodes verwerkt in volgorde van afstand tot de bron.

### AI-toepassingen van BFS

- **Vriendsuggesties op sociale netwerken** — mensen vinden binnen 2–3 verbindingen.
- **Webcrawlers** — pagina's laag voor laag indexeren vanaf een seed-URL.
- **Puzzeloplossers** — het minimale aantal zetten vinden (bijv. Rubik's cube-toestanden).

## Depth-First Search (DFS)

DFS gaat **zo diep mogelijk** voordat het terugkeert. Het gebruikt een **stack** (of recursie).

```
function dfs(graph, node, visited):
    visited.add(node)
    process(node)
    for neighbour in graph[node]:
        if neighbour not in visited:
            dfs(graph, neighbour, visited)
```

### Toepassingen van DFS

- **Topological sort** — taken ordenen zodat elke dependency eerst komt (build-systemen, cursusvoorwaarden).
- **Cycle detection** — als DFS een node herbezoekt die momenteel op de call stack staat, bestaat er een cycle.
- **Doolhof oplossen** — één pad volledig verkennen voordat alternatieven worden geprobeerd.
- **Connected components** — DFS uitvoeren vanaf elke onbezochte node om afzonderlijke clusters te identificeren.

## Dijkstra's Algorithm — Kortste pad met weights

Wanneer edges verschillende kosten hebben, volstaat BFS niet meer. Dijkstra's algorithm vindt het kortste gewogen pad van een bron naar alle andere nodes.

**Kernidee:** Breid altijd de **goedkoopste onbezochte node** uit met een min-heap (priority queue).

```
function dijkstra(graph, source):
    dist = {source: 0}          // all others: infinity
    pq = MinHeap([(0, source)])
    while pq is not empty:
        (cost, node) = pq.extractMin()
        if cost > dist[node]: continue
        for (neighbour, weight) in graph[node]:
            newDist = cost + weight
            if newDist < dist.getOrDefault(neighbour, INF):
                dist[neighbour] = newDist
                pq.insert((newDist, neighbour))
    return dist
```

**Tijdscomplexiteit:** O((V + E) log V) met een binaire heap.

<FunFact>Google Maps draait geen gewoon Dijkstra op het volledige wegennetwerk. Het gebruikt contraction hierarchies en A* search — maar beide bouwen voort op Dijkstra's fundamentele inzicht van greedy expansion op basis van de kortst bekende afstand.</FunFact>

### AI-toepassingen van Dijkstra

- **Routeplanning** — navigatie-apps die de snelste route vinden.
- **Netwerkrouting** — internetpakketten die het pad met de laagste latentie kiezen.
- **Knowledge graphs** — de kortste semantische verbinding tussen concepten vinden.

## Cycle Detection en Connected Components

### Cycle Detection

In een **directed graph** volg je nodes in drie toestanden: onbezocht, in-progress (op het huidige DFS-pad) en voltooid. Een in-progress node opnieuw tegenkomen betekent dat er een cycle bestaat.

In een **undirected graph** bestaat een cycle als DFS een bezochte node bereikt die niet de directe parent is.

### Connected Components

Voer BFS of DFS uit vanaf elke onbezochte node. Elke uitvoering ontdekt één connected component. In sociale netwerken onthullen components geïsoleerde gemeenschappen.

## Belangrijkste inzichten

1. **BFS** verkent niveau voor niveau — ideaal voor kortste paden in ongewogen graphs en op buren gebaseerde ontdekking.
2. **DFS** duikt diep — de ruggengraat van topological sorting, cycle detection en componentanalyse.
3. **Dijkstra** breidt BFS uit naar gewogen graphs met een priority queue — de basis van moderne routeplanning.
4. Graph-algoritmen zijn de motor achter kaarten, sociale netwerken, package managers en knowledge graphs.

<FunFact>Het World Wide Web bevat meer dan 50 miljard geïndexeerde pagina's. Zoekmachinecrawlers gebruiken varianten van BFS om pagina's te ontdekken en opnieuw te indexeren, met prioriteit op basis van pagina-importantie (PageRank).</FunFact>
