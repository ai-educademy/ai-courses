## Wat is een trie?

Een **trie** (uitgesproken als "try") is een boomachtige structuur waarin elke node één enkel karakter voorstelt. Paden van de root naar gemarkeerde nodes spellen opgeslagen woorden. In tegenstelling tot hash maps kun je met tries *alle woorden met een gemeenschappelijke prefix* opvragen zonder elke key te doorlopen.

```
        root
       / | \
      a   b   c
     /     \
    p       a
   / \       \
  p   e      t
  |
  l
  e
```

Elk pad codeert een woord: **apple**, **ape**, **bat**, **cat**.

## De TrieNode-klasse

Elke node bevat een map van children en een vlag die het einde van een woord aangeeft.

```python
class TrieNode:
    def __init__(self):
        self.children = {}
        self.is_end = False
```

Een volledige trie omhult dit met insert-, search- en prefix-operaties.

## Kernoperaties

### Insert — O(m) waarbij m = woordlengte

Loop karakter voor karakter, maak nodes aan waar nodig en markeer de laatste node.

```python
def insert(self, word):
    node = self.root
    for ch in word:
        if ch not in node.children:
            node.children[ch] = TrieNode()
        node = node.children[ch]
    node.is_end = True
```

### Search en startsWith

**Search** volgt het pad en controleert `is_end`. **startsWith** controleert alleen of het pad bestaat — het vereist geen `is_end = True`.

## Autocomplete met DFS

Om suggesties voor een prefix te geven, navigeer je naar de prefix-node en voer je vervolgens een DFS uit die alle onderliggende woorden verzamelt.

```python
def autocomplete(self, prefix):
    node = self._find(prefix)
    if not node:
        return []
    results = []
    self._dfs(node, prefix, results)
    return results
```

De `_dfs`-helper doorloopt recursief de children en voegt toe aan de resultaten wanneer `is_end` `True` is.

## Wildcard search — Woordenboek

LeetCode's **Design Add and Search Words Data Structure** vraagt je om `.` als wildcard te behandelen die elk willekeurig karakter matcht.

Wanneer je tijdens het zoeken een `.` tegenkomt, vertakt je naar *elke* child en ga je recursief verder. Dit verandert een eenvoudige traversal in een begrensde backtracking search.

```python
def search_wild(self, node, word, i):
    if i == len(word):
        return node.is_end
    if word[i] == '.':
        return any(
            self.search_wild(c, word, i + 1)
            for c in node.children.values()
        )
    if word[i] not in node.children:
        return False
    return self.search_wild(
        node.children[word[i]], word, i + 1
    )
```

## Langste gemeenschappelijke prefix

Doorloop de trie vanaf de root. De langste gemeenschappelijke prefix eindigt bij de eerste node met meer dan één child of een `is_end`-vlag.

## IP-routing met tries

Routers slaan IP-prefixen op in een **binary trie** — elke bit (0 of 1) vormt een branch. Longest prefix match vindt de meest specifieke route. Zo vinden je pakketten hun weg over het internet.

<FunFact>Backbone-routers op het internet voeren miljarden longest-prefix-match lookups per seconde uit met hardware-geoptimaliseerde tries die zijn ingebouwd in gespecialiseerde chips genaamd TCAMs.</FunFact>

## Compressed tries (radix trees)

Standaard tries verspillen ruimte aan ketens met slechts één child. Een **radix tree** voegt deze ketens samen tot enkele edges met multi-karakter labels.

```
Standard:  r → u → n → n → i → n → g
Radix:     r → "unning"
```

Dit vermindert het aantal nodes drastisch en wordt gebruikt door Linux-kernel routingtabellen en HTTP-routers zoals Go's `httprouter`.

## Tijd- en ruimtecomplexiteit

| Operatie       | Trie   | Hash Map |
|----------------|--------|----------|
| Insert         | O(m)   | O(m)     |
| Search         | O(m)   | O(m)     |
| Prefix query   | O(m+k) | O(n·m)   |

Hierbij is *m* de woordlengte, *k* het aantal resultaten en *n* het totale aantal keys. Tries domineren bij prefix-intensieve workloads.

## Wanneer kies je voor een trie?

- Prefix-gebaseerde queries (autocomplete, type-ahead)
- Wildcard- of regex-achtige matching op strings
- Langste gemeenschappelijke prefix in een verzameling
- IP-routing / longest prefix match
- Spellingcontrole met edit-distance variaties
