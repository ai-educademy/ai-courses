## Het grote idee

Sommige problemen lijken onmogelijk complex totdat je merkt dat ze bestaan uit **kleinere, herhalende deelproblemen**. Dynamic programming (DP) lost elk deelprobleem **één keer** op, slaat het resultaat op en hergebruikt het — een beetje geheugen inruilen voor een enorme versnelling.

Twee voorwaarden wijzen op een DP-kans:

1. **Overlappende deelproblemen** — dezelfde berekeningen komen steeds opnieuw voor.
2. **Optimale substructuur** — de beste oplossing voor het gehele probleem kan worden opgebouwd uit de beste oplossingen van de onderdelen.

## Fibonacci: het klassieke voorbeeld

### Naïeve recursie — O(2ⁿ)

```
function fib(n):
    if n <= 1: return n
    return fib(n - 1) + fib(n - 2)
```

Het aanroepen van `fib(5)` veroorzaakt **15** functie-aanroepen, waarvan vele overbodig. Bij `fib(40)` duurt het seconden; bij `fib(100)` zou het langer duren dan het universum bestaat.

### Memoisation (Top-Down) — O(n)

Sla resultaten op in een cache. Als je een deelprobleem al hebt opgelost, geef dan het gecachte antwoord terug.

```
memo = {}
function fib(n):
    if n in memo: return memo[n]
    if n <= 1: return n
    memo[n] = fib(n - 1) + fib(n - 2)
    return memo[n]
```

### Tabulation (Bottom-Up) — O(n)

Bouw het antwoord iteratief op vanuit de basisgevallen.

```
function fib(n):
    table = [0, 1]
    for i in 2..n:
        table[i] = table[i - 1] + table[i - 2]
    return table[n]
```

## Klassieke DP-problemen

### Trappen klimmen

Je kunt 1 of 2 treden tegelijk nemen. Hoeveel verschillende manieren zijn er om trede _n_ te bereiken?

```
function climbStairs(n):
    if n <= 2: return n
    a = 1, b = 2
    for i in 3..n:
        a, b = b, a + b
    return b
```

Dit is Fibonacci in vermomming — het antwoord voor trede _n_ is de som van de manieren om treden _n−1_ en _n−2_ te bereiken.

### Coin Change

Gegeven muntwaarden, vind het **minimum aantal munten** om een doelbedrag te bereiken.

```
function coinChange(coins, amount):
    dp = array of size (amount + 1), filled with INF
    dp[0] = 0
    for i in 1..amount:
        for coin in coins:
            if coin <= i:
                dp[i] = min(dp[i], dp[i - coin] + 1)
    return dp[amount] if dp[amount] != INF else -1
```

### Longest Common Subsequence (LCS)

Vind de langste reeks tekens die twee strings gemeen hebben (niet noodzakelijk aaneengesloten).

```
function lcs(s1, s2):
    m = s1.length, n = s2.length
    dp = 2D array of size (m+1) x (n+1), filled with 0
    for i in 1..m:
        for j in 1..n:
            if s1[i-1] == s2[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    return dp[m][n]
```

<FunFact>De `diff`-tool die verschillen tussen bestandsversies toont, is gebaseerd op LCS. Git gebruikt er een variant van telkens wanneer je `git diff` uitvoert.</FunFact>

## Echte AI-toepassingen

| Probleem | AI-domein | DP-techniek |
|----------|----------|------------|
| **Edit distance** (Levenshtein) | Spellingcontrole, fuzzy search | 2D-tabel die tekens vergelijkt |
| **Viterbi-algoritme** | Spraakherkenning, POS-tagging | Beste pad door een hidden Markov model |
| **Sequentie-alignment** | Bio-informatica (DNA-matching) | LCS-variant met gap-penalties |
| **CTC-decodering** | Handschrift / speech-to-text | Herhaalde tekens optimaal samenvoegen |
| **Beam search pruning** | Machinevertaling | Top-k gedeeltelijke vertalingen bewaren |

## Top-Down vs Bottom-Up

| Aspect | Top-Down (Memoisation) | Bottom-Up (Tabulation) |
|--------|----------------------|----------------------|
| Stijl | Recursief + cache | Iteratief + tabel |
| Opgeloste deelproblemen | Alleen de benodigde | Alle deelproblemen tot _n_ |
| Risico op stack overflow | Ja, bij diepe recursie | Nee |
| Gemak van optimalisatie | Moeilijker om ruimte te verminderen | Makkelijker (vaak volstaat alleen de laatste rij) |

**Vuistregel:** Begin met memoisation om de recurrentie te begrijpen en converteer vervolgens naar tabulation als je ruimte wilt optimaliseren of stacklimieten wilt vermijden.

## Belangrijkste inzichten

1. DP transformeert exponentiële brute-force naar polynomiale tijd door **herhaald werk te cachen**.
2. Identificeer overlappende deelproblemen en optimale substructuur voordat je voor DP kiest.
3. Memoisation is intuïtief; tabulation is efficiënt — leer beide.
4. Edit distance, Viterbi en sequentie-alignment tonen DP in het hart van moderne AI.
