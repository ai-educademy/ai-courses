## Wat maakt een stack monotoon?

Een **monotonic stack** houdt elementen in strikt stijgende of dalende volgorde van onder naar boven. Voordat je een nieuw element pusht, pop je alles wat de invariant schendt. Dit ene idee maakt O(n)-oplossingen mogelijk voor problemen die O(n²) lijken te vereisen.

```
Decreasing stack processing [2, 1, 5, 3, 4]:

Push 2 → [2]
Push 1 → [2, 1]         (1 < 2, invariant holds)
Pop 1, Pop 2, Push 5 → [5]  (5 > both)
Push 3 → [5, 3]
Pop 3, Push 4 → [5, 4]  (4 > 3)
```

## Next Greater Element

Zoek voor elk element het eerste grotere element rechts ervan. Brute force scant naar rechts voor elke positie - O(n²). Een monotoon dalende stack doet het in één doorgang.

```python
def next_greater(nums):
    result = [-1] * len(nums)
    stack = []
    for i, num in enumerate(nums):
        while stack and nums[stack[-1]] < num:
            result[stack.pop()] = num
        stack.append(i)
    return result
```

Wanneer we index `j` poppen, is het huidige element zijn next greater - gegarandeerd omdat alles ertussen kleiner was (ze waren eerder gepopt of blokkeerden `j` nooit).

## Largest Rectangle in Histogram

LeetCode's **Largest Rectangle in Histogram** is het kroonjuweel van monotonic stack-problemen.

**Kernidee:** zoek voor elke balk hoe ver die naar links en rechts kan uitstrekken als de kortste balk. Gebruik een stijgende stack - wanneer een balk gepopt wordt, definieert de poppende balk de rechtergrens, en de nieuwe stack top definieert de linkergrens.

```python
def largest_rectangle(heights):
    stack, max_area = [], 0
    for i, h in enumerate(heights + [0]):
        while stack and heights[stack[-1]] > h:
            height = heights[stack.pop()]
            width = i if not stack else i - stack[-1] - 1
            max_area = max(max_area, height * width)
        stack.append(i)
    return max_area
```

De sentinel `[0]` aan het einde dwingt de verwerking van alle resterende balken af.

## Trapping Rain Water - Stack-aanpak

In plaats van de klassieke two-pointer-methode berekent een monotoon dalende stack het opgevangen water laag voor laag. Wanneer een hogere balk verschijnt, pop je kortere balken en bereken je het water dat gevangen zit tussen de gepopte balk, de huidige balk en de nieuwe stack top.

```
Heights:  [3, 1, 2, 1, 3]
Water:       [2, 1, 2]  → total = 5
```

<FunFact>Trapping Rain Water heeft minstens vier verschillende O(n)-oplossingen - two-pointer, monotonic stack, prefix max-arrays en een geoptimaliseerde single-pass-aanpak. Interviewers zien graag meerdere strategieën.</FunFact>

## Monotonic Queue - Sliding Window Maximum

Een **monotonic deque** (dubbelzijdige queue) houdt een dalende volgorde aan en ondersteunt efficiënte verwijdering aan beide uiteinden. Voor **Sliding Window Maximum** (LeetCode 239):

1. Voordat je index `i` toevoegt, pop alle achterste elementen kleiner dan `nums[i]`
2. Push `i` naar de achterkant
3. De voorkant is altijd het maximum van het huidige venster
4. Verwijder de voorkant als die buiten het venster valt

```python
from collections import deque

def max_sliding_window(nums, k):
    dq, result = deque(), []
    for i, num in enumerate(nums):
        while dq and nums[dq[-1]] <= num:
            dq.pop()
        dq.append(i)
        if dq[0] <= i - k:
            dq.popleft()
        if i >= k - 1:
            result.append(nums[dq[0]])
    return result
```

Elk element komt hoogstens één keer in de deque en verlaat deze hoogstens één keer - **O(n)** totaal.

## Het patroon - Invariant behouden tijdens verwerking

Elk monotonic stack/queue-probleem volgt één sjabloon:

1. **Definieer de invariant** (stijgend of dalend)
2. **Pop overtreders** - en haal nuttige informatie uit elke pop
3. **Push het huidige element**
4. Het *pop-event* is waar het antwoord berekend wordt
