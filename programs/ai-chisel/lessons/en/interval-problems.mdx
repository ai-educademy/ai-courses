## Interval Representation

An interval is a pair `[start, end]`. Most interval problems begin with one critical step: **sort by start time**. This transforms a chaotic list into a sequence you can process left to right with simple comparisons.

```
Unsorted: [3,6] [1,4] [8,10] [2,5]
Sorted:   [1,4] [2,5] [3,6] [8,10]
```

## Merge Overlapping Intervals

The foundational interval problem (LeetCode 56). After sorting, two consecutive intervals overlap if the current start ≤ the previous end.

```python
def merge(intervals):
    intervals.sort()
    merged = [intervals[0]]
    for start, end in intervals[1:]:
        if start <= merged[-1][1]:
            merged[-1][1] = max(merged[-1][1], end)
        else:
            merged.append([start, end])
    return merged
```

Walk through `[1,4] [2,5] [3,6] [8,10]`:
- `[2,5]` overlaps `[1,4]` → merge to `[1,5]`
- `[3,6]` overlaps `[1,5]` → merge to `[1,6]`
- `[8,10]` no overlap → new interval

Result: `[1,6] [8,10]`

## Insert Interval

Given sorted non-overlapping intervals and a new interval, insert it and merge as needed (LeetCode 57). Split into three phases:

1. Add all intervals that end before the new one starts
2. Merge all overlapping intervals with the new one
3. Add remaining intervals unchanged

## Meeting Rooms I - Can Attend All?

Sort by start time. If any interval's start is before the previous interval's end, there is a conflict.

```python
def can_attend(intervals):
    intervals.sort()
    for i in range(1, len(intervals)):
        if intervals[i][0] < intervals[i - 1][1]:
            return False
    return True
```

## Meeting Rooms II - Minimum Rooms

This is where the **event-based approach** shines. Separate starts and ends into two sorted lists, then sweep:

```python
import heapq

def min_rooms(intervals):
    intervals.sort()
    heap = []
    for start, end in intervals:
        if heap and heap[0] <= start:
            heapq.heappop(heap)
        heapq.heappush(heap, end)
    return len(heap)
```

The heap tracks the earliest ending meeting. If it ends before the next one starts, we reuse the room. The heap's maximum size is the answer.

## Non-Overlapping Intervals - Minimum Removals

Given intervals, find the minimum number to remove so none overlap (LeetCode 435). Greedy: sort by **end time**, always keep the interval that ends earliest - this leaves the most room for future intervals.

```python
def erase_overlap(intervals):
    intervals.sort(key=lambda x: x[1])
    count, prev_end = 0, float('-inf')
    for start, end in intervals:
        if start >= prev_end:
            prev_end = end
        else:
            count += 1
    return count
```

## Interval List Intersections

Given two sorted interval lists, find all intersections (LeetCode 986). Use two pointers - one per list. The intersection of `[a1, a2]` and `[b1, b2]` is `[max(a1, b1), min(a2, b2)]` if it is valid (start ≤ end). Advance the pointer whose interval ends first.

## The Sweep Line Technique

Instead of comparing intervals pairwise, decompose each into events:

```
[1,4] → +1 at 1, -1 at 4
[2,6] → +1 at 2, -1 at 6
```

Sort events by time. Sweep left to right, maintaining a running count. The peak count answers "maximum overlap" questions. This generalises Meeting Rooms II to arbitrary event-processing problems.

## Calendar Booking

**My Calendar I** (LeetCode 729): maintain booked intervals and reject overlaps. **My Calendar II**: allow single overlaps but reject triple-bookings. The sweep line makes multi-layer overlap detection straightforward.

## Pattern Recognition - When to Suspect Intervals

Look for these clues in problem statements:

- Words like "overlapping", "conflicting", "schedule", "range"
- Pairs of start/end values
- Questions about "how many simultaneous" or "minimum resources"
- Data that can be represented as segments on a number line
