## Graphs Model the Connected World

Friendships, road networks, hyperlinks, neural pathways — whenever entities are linked by relationships, you have a **graph**. Formally, a graph G = (V, E) consists of vertices (nodes) and edges (connections).

Graphs can be **directed** (one-way streets) or **undirected** (two-way), and edges may carry **weights** (distances, costs, latencies).

## Representing Graphs

Two common representations:

| Representation | Space | Edge lookup | Best for |
|---------------|-------|-------------|----------|
| **Adjacency list** | O(V + E) | O(degree) | Sparse graphs (most real-world networks) |
| **Adjacency matrix** | O(V²) | O(1) | Dense graphs, quick edge existence checks |

```
// Adjacency list example
graph = {
    A: [(B, 4), (C, 1)],
    B: [(D, 1)],
    C: [(B, 2), (D, 5)],
    D: []
}
```

## Breadth-First Search (BFS)

BFS explores **level by level**, visiting all neighbours before moving deeper. It uses a **queue**.

```
function bfs(graph, start):
    visited = {start}
    queue = [start]
    while queue is not empty:
        node = queue.dequeue()
        process(node)
        for neighbour in graph[node]:
            if neighbour not in visited:
                visited.add(neighbour)
                queue.enqueue(neighbour)
```

**Key property:** BFS finds the **shortest path** in unweighted graphs because it processes nodes in order of distance from the source.

### AI Use Cases for BFS

- **Social network friend suggestions** — find people within 2–3 connections.
- **Web crawlers** — index pages layer by layer from a seed URL.
- **Puzzle solvers** — find the minimum number of moves (e.g. Rubik's cube states).

## Depth-First Search (DFS)

DFS goes **as deep as possible** before backtracking. It uses a **stack** (or recursion).

```
function dfs(graph, node, visited):
    visited.add(node)
    process(node)
    for neighbour in graph[node]:
        if neighbour not in visited:
            dfs(graph, neighbour, visited)
```

### Applications of DFS

- **Topological sort** — order tasks so every dependency comes first (build systems, course prerequisites).
- **Cycle detection** — if DFS revisits a node currently on the call stack, a cycle exists.
- **Maze solving** — explore one path fully before trying alternatives.
- **Connected components** — run DFS from every unvisited node to identify separate clusters.

## Dijkstra's Algorithm — Shortest Path with Weights

When edges have different costs, BFS no longer suffices. Dijkstra's algorithm finds the shortest weighted path from a source to all other nodes.

**Core idea:** Always expand the **cheapest unvisited node** using a min-heap (priority queue).

```
function dijkstra(graph, source):
    dist = {source: 0}          // all others: infinity
    pq = MinHeap([(0, source)])
    while pq is not empty:
        (cost, node) = pq.extractMin()
        if cost > dist[node]: continue
        for (neighbour, weight) in graph[node]:
            newDist = cost + weight
            if newDist < dist.getOrDefault(neighbour, INF):
                dist[neighbour] = newDist
                pq.insert((newDist, neighbour))
    return dist
```

**Time complexity:** O((V + E) log V) with a binary heap.

<FunFact>Google Maps does not run plain Dijkstra on the entire road network. It uses contraction hierarchies and A* search — but both build on Dijkstra's foundational insight of greedy expansion by shortest known distance.</FunFact>

### AI Use Cases for Dijkstra

- **Route planning** — navigation apps finding the fastest drive.
- **Network routing** — internet packets choosing the lowest-latency path.
- **Knowledge graphs** — finding the shortest semantic connection between concepts.

## Cycle Detection and Connected Components

### Cycle Detection

In a **directed graph**, track nodes in three states: unvisited, in-progress (on the current DFS path), and completed. Encountering an in-progress node means a cycle exists.

In an **undirected graph**, a cycle exists if DFS reaches a visited node that is not the immediate parent.

### Connected Components

Run BFS or DFS from each unvisited node. Each run discovers one connected component. In social networks, components reveal isolated communities.

## Key Takeaways

1. **BFS** explores level by level — ideal for shortest paths in unweighted graphs and neighbour-based discovery.
2. **DFS** dives deep — the backbone of topological sorting, cycle detection, and component analysis.
3. **Dijkstra** extends BFS to weighted graphs using a priority queue — the foundation of modern route planning.
4. Graph algorithms are the engine behind maps, social networks, package managers, and knowledge graphs.

<FunFact>The World Wide Web contains over 50 billion indexed pages. Search engine crawlers use variations of BFS to discover and re-index pages, prioritising by page importance (PageRank).</FunFact>
