## What Is a Trie?

A **trie** (pronounced "try") is a tree-like structure where each node represents a single character. Paths from root to marked nodes spell out stored words. Unlike hash maps, tries let you query *all words sharing a prefix* without scanning every key.

```
        root
       / | \
      a   b   c
     /     \
    p       a
   / \       \
  p   e      t
  |
  l
  e
```

Every path encodes a word: **apple**, **ape**, **bat**, **cat**.

## The TrieNode Class

Each node holds a map of children and a flag indicating word completion.

```python
class TrieNode:
    def __init__(self):
        self.children = {}
        self.is_end = False
```

A full Trie wraps this with insert, search, and prefix operations.

## Core Operations

### Insert — O(m) where m = word length

Walk character by character, creating nodes as needed, then mark the last node.

```python
def insert(self, word):
    node = self.root
    for ch in word:
        if ch not in node.children:
            node.children[ch] = TrieNode()
        node = node.children[ch]
    node.is_end = True
```

### Search and startsWith

**Search** follows the path and checks `is_end`. **startsWith** only checks whether the path exists — it does not require `is_end = True`.

## Autocomplete with DFS

To suggest completions for a prefix, navigate to the prefix node, then run DFS collecting all words beneath it.

```python
def autocomplete(self, prefix):
    node = self._find(prefix)
    if not node:
        return []
    results = []
    self._dfs(node, prefix, results)
    return results
```

The `_dfs` helper recurses through children, appending to results whenever `is_end` is `True`.

## Wildcard Search — Word Dictionary

LeetCode's **Design Add and Search Words Data Structure** asks you to handle `.` as a wildcard matching any single character.

During search, when you hit a `.`, branch into *every* child and continue recursively. This turns a simple traversal into a bounded backtracking search.

```python
def search_wild(self, node, word, i):
    if i == len(word):
        return node.is_end
    if word[i] == '.':
        return any(
            self.search_wild(c, word, i + 1)
            for c in node.children.values()
        )
    if word[i] not in node.children:
        return False
    return self.search_wild(
        node.children[word[i]], word, i + 1
    )
```

## Longest Common Prefix

Walk the trie from the root. The longest common prefix ends at the first node with more than one child or an `is_end` flag.

## IP Routing with Tries

Routers store IP prefixes in a **binary trie** — each bit (0 or 1) forms a branch. Longest prefix match finds the most specific route. This is how your packets find their way across the internet.

<FunFact>Internet backbone routers perform billions of longest-prefix-match lookups per second using hardware-optimised tries built into specialised chips called TCAMs.</FunFact>

## Compressed Tries (Radix Trees)

Standard tries waste space on single-child chains. A **radix tree** merges these chains into single edges storing multi-character labels.

```
Standard:  r → u → n → n → i → n → g
Radix:     r → "unning"
```

This cuts node count dramatically and is used by Linux kernel routing tables and HTTP routers like Go's `httprouter`.

## Time and Space Complexity

| Operation   | Trie   | Hash Map |
|-------------|--------|----------|
| Insert      | O(m)   | O(m)     |
| Search      | O(m)   | O(m)     |
| Prefix query| O(m+k) | O(n·m)   |

Here *m* is word length, *k* is results count, and *n* is total keys. Tries dominate for prefix-heavy workloads.

## When to Reach for a Trie

- Prefix-based queries (autocomplete, type-ahead)
- Wildcard or regex-like matching on strings
- Longest common prefix across a set
- IP routing / longest prefix match
- Spell checking with edit-distance variations
