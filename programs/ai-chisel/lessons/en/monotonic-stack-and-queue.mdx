## What Makes a Stack Monotonic?

A **monotonic stack** maintains elements in strictly increasing or decreasing order from bottom to top. Before pushing a new element, you pop everything that violates the invariant. This single idea unlocks O(n) solutions for problems that seem to require O(n²).

```
Decreasing stack processing [2, 1, 5, 3, 4]:

Push 2 → [2]
Push 1 → [2, 1]         (1 < 2, invariant holds)
Pop 1, Pop 2, Push 5 → [5]  (5 > both)
Push 3 → [5, 3]
Pop 3, Push 4 → [5, 4]  (4 > 3)
```

## Next Greater Element

For each element, find the first larger element to its right. Brute force scans rightward for each position — O(n²). A monotonic decreasing stack does it in one pass.

```python
def next_greater(nums):
    result = [-1] * len(nums)
    stack = []
    for i, num in enumerate(nums):
        while stack and nums[stack[-1]] < num:
            result[stack.pop()] = num
        stack.append(i)
    return result
```

When we pop index `j`, the current element is its next greater — guaranteed because everything between was smaller (they were popped earlier or never blocked `j`).

## Daily Temperatures (LeetCode 739)

*Given daily temperatures, find how many days until a warmer day.* This is next-greater-element with indices — the answer for index `j` is `i − j` when `j` gets popped by index `i`.

## Stock Span Problem

*For each day, how many consecutive days (including today) had price ≤ today's price?* Use a monotonic decreasing stack storing `(price, index)`. When you pop, the gap between the current index and the new stack top gives the span.

## Largest Rectangle in Histogram

LeetCode's **Largest Rectangle in Histogram** is the crown jewel of monotonic stack problems.

**Key insight:** for each bar, find how far left and right it can extend as the shortest bar. Use an increasing stack — when a bar gets popped, the popping bar defines its right boundary, and the new stack top defines its left boundary.

```python
def largest_rectangle(heights):
    stack, max_area = [], 0
    for i, h in enumerate(heights + [0]):
        while stack and heights[stack[-1]] > h:
            height = heights[stack.pop()]
            width = i if not stack else i - stack[-1] - 1
            max_area = max(max_area, height * width)
        stack.append(i)
    return max_area
```

The sentinel `[0]` at the end forces all remaining bars to be processed.

## Trapping Rain Water — Stack Approach

Instead of the classic two-pointer method, a monotonic decreasing stack computes trapped water layer by layer. When a taller bar appears, pop shorter bars and calculate the water trapped between the popped bar, the current bar, and the new stack top.

```
Heights:  [3, 1, 2, 1, 3]
Water:       [2, 1, 2]  → total = 5
```

<FunFact>Trapping Rain Water has at least four distinct O(n) solutions — two-pointer, monotonic stack, prefix max arrays, and a single-pass optimised approach. Interviewers love seeing multiple strategies.</FunFact>

## Monotonic Queue — Sliding Window Maximum

A **monotonic deque** (double-ended queue) maintains decreasing order and supports efficient removal from both ends. For **Sliding Window Maximum** (LeetCode 239):

1. Before adding index `i`, pop all back elements smaller than `nums[i]`
2. Push `i` to the back
3. The front is always the current window's maximum
4. Remove the front if it falls outside the window

```python
from collections import deque

def max_sliding_window(nums, k):
    dq, result = deque(), []
    for i, num in enumerate(nums):
        while dq and nums[dq[-1]] <= num:
            dq.pop()
        dq.append(i)
        if dq[0] <= i - k:
            dq.popleft()
        if i >= k - 1:
            result.append(nums[dq[0]])
    return result
```

Each element enters and leaves the deque at most once — **O(n)** total.

## The Pattern — Maintain Invariant While Processing

Every monotonic stack/queue problem follows one template:

1. **Define the invariant** (increasing or decreasing)
2. **Pop violators** — and extract useful information from each pop
3. **Push the current element**
4. The *pop event* is where the answer is computed
