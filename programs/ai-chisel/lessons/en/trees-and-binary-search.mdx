## Trees Are Everywhere in AI

Every time a spam filter classifies an email or a database retrieves your order in milliseconds, a tree data structure is doing the heavy lifting. Trees model **hierarchical decisions** - each node narrows the search space until you reach an answer.

<Illustration src="/images/lessons/trees-and-binary-search.svg" alt="A binary search tree with nodes labelled by value, showing left-smaller and right-larger property" caption="In a BST every left child is smaller and every right child is larger than its parent." />

## Binary Search Trees (BSTs)

A BST stores elements so that for any node, all values in its **left** subtree are smaller and all values in its **right** subtree are larger.

### Search

```
function search(node, target):
    if node is null: return null
    if target == node.value: return node
    if target < node.value: return search(node.left, target)
    return search(node.right, target)
```

Each comparison eliminates roughly half the remaining nodes - O(log n) on a balanced tree.

### Insertion

```
function insert(node, value):
    if node is null: return new Node(value)
    if value < node.value:
        node.left = insert(node.left, value)
    else:
        node.right = insert(node.right, value)
    return node
```

### Deletion

Deleting is the trickiest operation. Three cases arise:

1. **Leaf node** - simply remove it.
2. **One child** - replace the node with its child.
3. **Two children** - swap the node's value with its in-order successor (smallest value in the right subtree), then delete that successor.

<Quiz question="What is the in-order successor of a node in a BST?" options="The parent of the node|The largest node in the left subtree|The smallest node in the right subtree|The root of the tree" answer="2" explanation="The in-order successor is the smallest value in the right subtree - found by going right once, then left as far as possible." />

## Balanced vs Unbalanced

If you insert sorted data into a plain BST, it degenerates into a linked list - O(n) per operation. **Self-balancing** trees fix this:

| Tree | Balance Rule | Typical Use |
|------|-------------|-------------|
| AVL | Height difference ≤ 1 between subtrees | In-memory lookups |
| Red-Black | Colour-based rotation rules | Language standard libraries (e.g. Java TreeMap) |
| B-Tree | Multiple keys per node, all leaves at same depth | Databases and file systems |

<FunFact>PostgreSQL and MySQL both use B-tree variants as their default index structure. A B-tree with branching factor 100 can index a billion rows in just five levels.</FunFact>

## Tree Traversals

Traversals visit every node exactly once. The three classic orders differ only in **when** you process the current node relative to its children.

| Traversal | Order | Use Case |
|-----------|-------|----------|
| **In-order** | Left → Node → Right | Produces sorted output from a BST |
| **Pre-order** | Node → Left → Right | Serialise / clone a tree |
| **Post-order** | Left → Right → Node | Safely delete a tree; evaluate expression trees |

```
function inOrder(node):
    if node is null: return
    inOrder(node.left)
    process(node)
    inOrder(node.right)
```

## Heaps and Priority Queues

A **heap** is a complete binary tree where every parent is greater (max-heap) or smaller (min-heap) than its children. It supports:

- **Insert** - add at the end, bubble up. O(log n).
- **Extract min/max** - remove the root, move the last element to the root, sift down. O(log n).

Priority queues built on heaps power:
- **Dijkstra's algorithm** (next lesson!) - always expand the nearest unvisited node.
- **Task scheduling** - OS schedulers pick the highest-priority process.
- **Beam search in NLP** - keep the top-k most promising sentence completions.

```
function siftUp(heap, index):
    while index > 0:
        parent = (index - 1) / 2
        if heap[index] < heap[parent]:
            swap(heap, index, parent)
            index = parent
        else: break
```

<Quiz question="What is the time complexity of finding the minimum element in a min-heap?" options="O(n)|O(log n)|O(1)|O(n log n)" answer="2" explanation="The minimum is always at the root of a min-heap, so accessing it is O(1) - no traversal needed." />

## Real AI Applications

- **Decision trees** in machine learning split data at each node using the feature that best separates classes - a direct application of tree structure.
- **Syntax trees** in NLP represent grammatical structure, enabling tasks like sentiment analysis and machine translation.
- **Monte Carlo Tree Search (MCTS)** powers game-playing AI such as AlphaGo, exploring possible moves as a tree of game states.
- **Huffman trees** compress data by assigning shorter codes to frequent symbols - used in model weight compression.

<FunFact>The scikit-learn decision tree classifier builds a tree where each internal node tests a feature threshold. A trained model with depth 20 can represent over a million distinct decision paths.</FunFact>

## Key Takeaways

1. BSTs provide O(log n) search, insert, and delete - but only when balanced.
2. Traversal order determines what information you extract from a tree.
3. Heaps give O(1) access to the extreme value and O(log n) insertion - ideal for priority-based AI algorithms.
4. Trees underpin decision-making across ML, databases, compilers, and game AI.
