## పెద్ద ఆలోచన

కొన్ని problems చూడటానికి చాలా complex గా కనిపిస్తాయి, కానీ జాగ్రత్తగా చూస్తే అవి **చిన్న, పునరావృతమయ్యే sub-problems** తో తయారయ్యాయని తెలుస్తుంది. Dynamic programming (DP) ప్రతి sub-problem ను **ఒక్కసారి** మాత్రమే solve చేసి, result ను store చేసి, దాన్ని reuse చేస్తుంది - కొంచెం memory ఖర్చు చేసి గొప్ప speed-up సాధిస్తుంది.

రెండు conditions DP అవకాశాన్ని సూచిస్తాయి:

1. **Overlapping sub-problems** - అదే calculations చాలాసార్లు పునరావృతమవుతాయి.
2. **Optimal substructure** - మొత్తం problem యొక్క best solution దాని భాగాల best solutions నుండి నిర్మించవచ్చు.

## Fibonacci: అత్యంత Classic ఉదాహరణ

### Naïve Recursion - O(2ⁿ)

```
function fib(n):
    if n <= 1: return n
    return fib(n - 1) + fib(n - 2)
```

`fib(5)` call చేస్తే **15** function calls జరుగుతాయి, వాటిలో చాలా redundant. `fib(40)` కి seconds పడుతుంది; `fib(100)` కి అయితే universe కంటే ఎక్కువ time పడుతుంది.

### Memoisation (Top-Down) - O(n)

Results ను ఒక cache లో store చేయండి. ఒక sub-problem ఇప్పటికే solve అయి ఉంటే, cached answer return చేయండి.

```
memo = {}
function fib(n):
    if n in memo: return memo[n]
    if n <= 1: return n
    memo[n] = fib(n - 1) + fib(n - 2)
    return memo[n]
```

### Tabulation (Bottom-Up) - O(n)

Base cases నుండి iteratively answer నిర్మించండి.

```
function fib(n):
    table = [0, 1]
    for i in 2..n:
        table[i] = table[i - 1] + table[i - 2]
    return table[n]
```

## Classic DP Problems

### మెట్లు ఎక్కడం

మీరు ఒకేసారి 1 లేదా 2 మెట్లు ఎక్కవచ్చు. Step _n_ కి చేరుకోవడానికి ఎన్ని విభిన్న మార్గాలు ఉన్నాయి?

```
function climbStairs(n):
    if n <= 2: return n
    a = 1, b = 2
    for i in 3..n:
        a, b = b, a + b
    return b
```

ఇది మారువేషంలో Fibonacci - step _n_ కి answer, step _n−1_ మరియు _n−2_ కి చేరుకునే మార్గాల sum.

### Coin Change

ఇచ్చిన coin denominations తో target amount చేయడానికి **అత్యల్ప coins** కనుగొనండి.

```
function coinChange(coins, amount):
    dp = array of size (amount + 1), filled with INF
    dp[0] = 0
    for i in 1..amount:
        for coin in coins:
            if coin <= i:
                dp[i] = min(dp[i], dp[i - coin] + 1)
    return dp[amount] if dp[amount] != INF else -1
```

### Longest Common Subsequence (LCS)

రెండు strings లో common characters యొక్క పొడవైన sequence కనుగొనండి (తప్పనిసరిగా contiguous కానవసరం లేదు).

```
function lcs(s1, s2):
    m = s1.length, n = s2.length
    dp = 2D array of size (m+1) x (n+1), filled with 0
    for i in 1..m:
        for j in 1..n:
            if s1[i-1] == s2[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    return dp[m][n]
```

<FunFact>File versions మధ్య changes చూపించే `diff` tool LCS పై ఆధారపడింది. మీరు `git diff` run చేసినప్పుడల్లా Git దాని variant ని వాడుతుంది.</FunFact>

<ThinkAboutIt>Coin change minimum coins సంఖ్య అడుగుతుంది. Amount చేయడానికి మొత్తం విభిన్న మార్గాల సంఖ్య లెక్కించాలంటే? Recurrence relation ఎలా మారుతుంది?</ThinkAboutIt>

## AI లో నిజమైన అనువర్తనాలు

| Problem | AI Domain | DP Technique |
|---------|-----------|-------------|
| **Edit distance** (Levenshtein) | Spell checkers, fuzzy search | Characters పోల్చే 2D table |
| **Viterbi algorithm** | Speech recognition, POS tagging | Hidden Markov model లో best path |
| **Sequence alignment** | Bioinformatics (DNA matching) | Gap penalties తో LCS variant |
| **CTC decoding** | Handwriting / speech-to-text | Repeated characters ను optimally merge చేయడం |
| **Beam search pruning** | Machine translation | Top-k partial translations ఉంచడం |

## Top-Down vs Bottom-Up

| అంశం | Top-Down (Memoisation) | Bottom-Up (Tabulation) |
|------|----------------------|----------------------|
| Style | Recursive + cache | Iterative + table |
| Solve అయిన sub-problems | అవసరమైనవి మాత్రమే | _n_ వరకు అన్ని sub-problems |
| Stack overflow risk | అవును, deep recursion లో | కాదు |
| Optimisation సౌలభ్యం | Space తగ్గించడం కష్టం | సులభం (తరచుగా last row మాత్రమే ఉంచితే చాలు) |

**సాధారణ నియమం:** ముందు memoisation తో recurrence అర్థం చేసుకోండి, తర్వాత space optimise చేయడానికి లేదా stack limits నివారించడానికి tabulation కి convert చేయండి.

<ThinkAboutIt>Coin change table size O(amount). LCS table size O(m × n). ఒకేసారి రెండు rows మాత్రమే ఉంచి LCS space ను O(min(m, n)) కి తగ్గించగలరా? ఏ information కోల్పోతారు?</ThinkAboutIt>

## ముఖ్యమైన అంశాలు

1. DP exponential brute-force ను polynomial time గా మారుస్తుంది **repeated work ను cache చేయడం** ద్వారా.
2. DP వాడే ముందు overlapping sub-problems మరియు optimal substructure గుర్తించండి.
3. Memoisation intuitive; tabulation efficient - రెండూ నేర్చుకోండి.
4. Edit distance, Viterbi మరియు sequence alignment modern AI హృదయంలో DP ని చూపిస్తాయి.
