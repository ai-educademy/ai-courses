## Trie అంటే ఏమిటి?

**Trie** (ఉచ్చారణ "try") అనేది ప్రతి node ఒక్క character ను represent చేసే tree లాంటి structure. Root నుండి marked nodes వరకు ఉన్న paths stored words ను spell చేస్తాయి. Hash maps కి భిన్నంగా, tries *ఒక prefix share చేసే అన్ని words* ను ప్రతి key scan చేయకుండా query చేయగలవు.

```
        root
       / | \
      a   b   c
     /     \
    p       a
   / \       \
  p   e      t
  |
  l
  e
```

ప్రతి path ఒక word ను encode చేస్తుంది: **apple**, **ape**, **bat**, **cat**.

## TrieNode Class

ప్రతి node లో children map మరియు word completion ని indicate చేసే flag ఉంటాయి.

```python
class TrieNode:
    def __init__(self):
        self.children = {}
        self.is_end = False
```

పూర్తి Trie దీనిని insert, search, మరియు prefix operations తో wrap చేస్తుంది.

## ప్రధాన Operations

### Insert — O(m) ఇక్కడ m = word యొక్క length

Character by character నడవండి, అవసరమైతే nodes సృష్టించండి, తర్వాత చివరి node ని mark చేయండి.

```python
def insert(self, word):
    node = self.root
    for ch in word:
        if ch not in node.children:
            node.children[ch] = TrieNode()
        node = node.children[ch]
    node.is_end = True
```

### Search మరియు startsWith

**Search** path ను follow చేసి `is_end` check చేస్తుంది. **startsWith** path ఉందా లేదా అని మాత్రమే check చేస్తుంది — దీనికి `is_end = True` అవసరం లేదు.

## DFS తో Autocomplete

ఒక prefix కోసం suggestions ఇవ్వడానికి, prefix node కి navigate చేసి, తర్వాత DFS run చేసి దాని కింద ఉన్న అన్ని words collect చేయండి.

```python
def autocomplete(self, prefix):
    node = self._find(prefix)
    if not node:
        return []
    results = []
    self._dfs(node, prefix, results)
    return results
```

`_dfs` helper children లో recursively traverse చేసి `is_end` `True` అయినప్పుడల్లా results కి append చేస్తుంది.

<ThinkAboutIt>Autocomplete ని top-k అత్యధిక frequent suggestions కి ఎలా limit చేస్తారు? ప్రతి node కి ఏ extra data అవసరం?</ThinkAboutIt>

## Wildcard Search — Word Dictionary

LeetCode యొక్క **Design Add and Search Words Data Structure** లో `.` ని ఏదైనా single character కి match అయ్యే wildcard గా handle చేయాలి.

Search సమయంలో `.` వచ్చినప్పుడు, *ప్రతి* child లోకి branch చేసి recursively continue చేయండి. ఇది simple traversal ని bounded backtracking search గా మారుస్తుంది.

```python
def search_wild(self, node, word, i):
    if i == len(word):
        return node.is_end
    if word[i] == '.':
        return any(
            self.search_wild(c, word, i + 1)
            for c in node.children.values()
        )
    if word[i] not in node.children:
        return False
    return self.search_wild(
        node.children[word[i]], word, i + 1
    )
```

## Longest Common Prefix

Root నుండి trie లో నడవండి. Longest common prefix మొదటి node వద్ద ముగుస్తుంది, ఆ node కి ఒకటి కంటే ఎక్కువ children ఉన్నప్పుడు లేదా `is_end` flag ఉన్నప్పుడు.

## Tries తో IP Routing

Routers IP prefixes ను **binary trie** లో store చేస్తారు — ప్రతి bit (0 లేదా 1) ఒక branch ఏర్పరుస్తుంది. Longest prefix match అత్యంత specific route ని కనుగొంటుంది. మీ packets internet లో తమ దారిని ఇలాగే కనుగొంటాయి.

<FunFact>Internet backbone routers TCAMs అనే specialised chips లో నిర్మించిన hardware-optimised tries ఉపయోగించి ప్రతి second బిలియన్ల longest-prefix-match lookups చేస్తాయి.</FunFact>

## Compressed Tries (Radix Trees)

Standard tries single-child chains పై space వృథా చేస్తాయి. **Radix tree** ఈ chains ని merge చేసి multi-character labels ఉన్న single edges గా మారుస్తుంది.

```
Standard:  r → u → n → n → i → n → g
Radix:     r → "unning"
```

ఇది node count ని గణనీయంగా తగ్గిస్తుంది మరియు Linux kernel routing tables మరియు Go యొక్క `httprouter` వంటి HTTP routers లో ఉపయోగించబడుతుంది.

## Time మరియు Space Complexity

| Operation    | Trie   | Hash Map |
|--------------|--------|----------|
| Insert       | O(m)   | O(m)     |
| Search       | O(m)   | O(m)     |
| Prefix query | O(m+k) | O(n·m)   |

ఇక్కడ *m* word length, *k* results count, మరియు *n* total keys. Prefix-heavy workloads లో tries ఆధిపత్యం చెలాయిస్తాయి.

## Trie ఎప్పుడు ఉపయోగించాలి

- Prefix-based queries (autocomplete, type-ahead)
- Strings పై wildcard లేదా regex-like matching
- ఒక set లో longest common prefix
- IP routing / longest prefix match
- Edit-distance variations తో spell checking
