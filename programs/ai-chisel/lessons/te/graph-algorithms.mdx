## Graphs అనుసంధానిత ప్రపంచాన్ని మోడల్ చేస్తాయి

స్నేహాలు, రోడ్ నెట్‌వర్క్‌లు, hyperlinks, నాడీ మార్గాలు - entities సంబంధాల ద్వారా అనుసంధానమైనప్పుడల్లా, మీకు ఒక **graph** ఉంటుంది. అధికారికంగా, ఒక graph G = (V, E) vertices (nodes) మరియు edges (connections) తో కూడి ఉంటుంది.

Graphs **directed** (ఏకదిశ వీధులు) లేదా **undirected** (ద్విదిశ) కావచ్చు, మరియు edges పై **weights** (దూరాలు, ఖర్చులు, latencies) ఉండవచ్చు.

## Graphs ను ఎలా Represent చేయాలి

రెండు సాధారణ representations:

| Representation | Space | Edge lookup | దేనికి అనువైనది |
|---------------|-------|-------------|-----------------|
| **Adjacency list** | O(V + E) | O(degree) | Sparse graphs (చాలావరకు real-world networks) |
| **Adjacency matrix** | O(V²) | O(1) | Dense graphs, వేగవంతమైన edge ఉనికి తనిఖీలు |

```
// Adjacency list example
graph = {
    A: [(B, 4), (C, 1)],
    B: [(D, 1)],
    C: [(B, 2), (D, 5)],
    D: []
}
```

## Breadth-First Search (BFS)

BFS **స్థాయి వారీగా** అన్వేషిస్తుంది - లోతుగా వెళ్ళే ముందు అన్ని పొరుగువారిని సందర్శిస్తుంది. ఇది ఒక **queue** ను ఉపయోగిస్తుంది.

```
function bfs(graph, start):
    visited = {start}
    queue = [start]
    while queue is not empty:
        node = queue.dequeue()
        process(node)
        for neighbour in graph[node]:
            if neighbour not in visited:
                visited.add(neighbour)
                queue.enqueue(neighbour)
```

**ముఖ్య లక్షణం:** BFS unweighted graphs లో **shortest path** కనుగొంటుంది ఎందుకంటే ఇది source నుండి దూరం క్రమంలో nodes ను ప్రాసెస్ చేస్తుంది.

### BFS యొక్క AI Use Cases

- **Social network friend suggestions** - 2–3 connections లోపల వ్యక్తులను కనుగొనడం.
- **Web crawlers** - seed URL నుండి పొర పొరగా pages ను index చేయడం.
- **Puzzle solvers** - కనీస సంఖ్యలో కదలికలు కనుగొనడం (ఉదా. Rubik's cube states).

## Depth-First Search (DFS)

DFS **సాధ్యమైనంత లోతుగా** వెళ్తుంది, తర్వాత backtrack చేస్తుంది. ఇది ఒక **stack** (లేదా recursion) ఉపయోగిస్తుంది.

```
function dfs(graph, node, visited):
    visited.add(node)
    process(node)
    for neighbour in graph[node]:
        if neighbour not in visited:
            dfs(graph, neighbour, visited)
```

### DFS యొక్క Applications

- **Topological sort** - ప్రతి dependency ముందు వచ్చేలా tasks ను క్రమం చేయడం (build systems, course prerequisites).
- **Cycle detection** - DFS ప్రస్తుతం call stack పై ఉన్న node ను మళ్ళీ సందర్శిస్తే, cycle ఉన్నట్లు.
- **Maze solving** - ప్రత్యామ్నాయాలను ప్రయత్నించే ముందు ఒక మార్గాన్ని పూర్తిగా అన్వేషించడం.
- **Connected components** - ప్రతి సందర్శించని node నుండి DFS నడపడం ద్వారా వేర్వేరు clusters ను గుర్తించడం.

<ThinkAboutIt>ఒక package manager dependencies ను సరైన క్రమంలో install చేయాలి. BFS ఉపయోగించాలా లేదా DFS-based topological sort ఉపయోగించాలా? dependency graph లో cycle ఉంటే ఏమి జరుగుతుంది?</ThinkAboutIt>

## Dijkstra's Algorithm - Weights తో Shortest Path

Edges కు వేర్వేరు costs ఉన్నప్పుడు, BFS సరిపోదు. Dijkstra's algorithm ఒక source నుండి అన్ని ఇతర nodes కు shortest weighted path కనుగొంటుంది.

**ప్రధాన ఆలోచన:** ఎల్లప్పుడూ min-heap (priority queue) ఉపయోగించి **అత్యంత చౌకైన unvisited node** ను విస్తరించండి.

```
function dijkstra(graph, source):
    dist = {source: 0}          // all others: infinity
    pq = MinHeap([(0, source)])
    while pq is not empty:
        (cost, node) = pq.extractMin()
        if cost > dist[node]: continue
        for (neighbour, weight) in graph[node]:
            newDist = cost + weight
            if newDist < dist.getOrDefault(neighbour, INF):
                dist[neighbour] = newDist
                pq.insert((newDist, neighbour))
    return dist
```

**Time complexity:** Binary heap తో O((V + E) log V).

<FunFact>Google Maps మొత్తం road network పై plain Dijkstra నడపదు. ఇది contraction hierarchies మరియు A* search ఉపయోగిస్తుంది - కానీ రెండూ Dijkstra యొక్క మౌలిక అంతర్దృష్టిపై ఆధారపడి ఉన్నాయి: shortest known distance ద్వారా greedy expansion.</FunFact>

### Dijkstra యొక్క AI Use Cases

- **Route planning** - అత్యంత వేగవంతమైన మార్గాన్ని కనుగొనే navigation apps.
- **Network routing** - అత్యల్ప latency path ఎంచుకునే internet packets.
- **Knowledge graphs** - concepts మధ్య shortest semantic connection కనుగొనడం.

## Cycle Detection మరియు Connected Components

### Cycle Detection

ఒక **directed graph** లో, nodes ను మూడు states లో track చేయండి: unvisited, in-progress (ప్రస్తుత DFS path పై), మరియు completed. In-progress node ను మళ్ళీ ఎదుర్కోవడం అంటే cycle ఉన్నట్లు.

ఒక **undirected graph** లో, DFS immediate parent కాని visited node ను చేరుకుంటే cycle ఉన్నట్లు.

### Connected Components

ప్రతి సందర్శించని node నుండి BFS లేదా DFS నడపండి. ప్రతి run ఒక connected component ను కనుగొంటుంది. Social networks లో, components ఒంటరి సముదాయాలను వెల్లడిస్తాయి.

<ThinkAboutIt>ఒక AI fraud detection system transactions ను graph గా మోడల్ చేస్తుంది, ఇక్కడ accounts nodes మరియు transfers edges. Connected components money-laundering rings ను గుర్తించడంలో ఎలా సహాయపడగలవు?</ThinkAboutIt>

## ముఖ్యమైన విషయాలు

1. **BFS** స్థాయి వారీగా అన్వేషిస్తుంది - unweighted graphs లో shortest paths మరియు neighbour-based discovery కు అనువైనది.
2. **DFS** లోతుగా వెళ్తుంది - topological sorting, cycle detection, మరియు component analysis కు వెన్నెముక.
3. **Dijkstra** priority queue ఉపయోగించి BFS ను weighted graphs కు విస్తరిస్తుంది - ఆధునిక route planning కు పునాది.
4. Graph algorithms maps, social networks, package managers, మరియు knowledge graphs వెనుక ఉన్న engine.

<FunFact>World Wide Web లో 50 billion కంటే ఎక్కువ indexed pages ఉన్నాయి. Search engine crawlers pages ను కనుగొని మళ్ళీ index చేయడానికి BFS variations ఉపయోగిస్తాయి, page importance (PageRank) ఆధారంగా ప్రాధాన్యత ఇస్తూ.</FunFact>
