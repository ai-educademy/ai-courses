## Hashing ఎందుకు ముఖ్యం

ఒక recommendation engine మీరు ఇప్పటికే ఒక సినిమా చూశారో లేదో తనిఖీ చేయాలనుకుంటే - తన catalogue లో లక్షల సినిమాల్లో - అది ఒక list ను scan చేయడం afford చేయలేదు. ఒక **hash table** ఆ ప్రశ్నకు O(1) సగటు సమయంలో సమాధానం ఇస్తుంది.

Hash tables production software లో అత్యధికంగా ఉపయోగించే data structure, మరియు AI systems feature lookups, caching మరియు deduplication కోసం వీటిపై ఎక్కువగా ఆధారపడతాయి.

## Hashing ఎలా పని చేస్తుంది

ఒక **hash function** ఒక input (string, number లేదా object) తీసుకుని ఒక integer return చేస్తుంది - అదే **hash code**. ఆ code ను modulo ద్వారా **bucket index** కు map చేస్తారు:

```
index = hashFunction(key) % numberOfBuckets
```

మంచి hash function యొక్క లక్షణాలు:
- **Deterministic** - ఒకే input ఎల్లప్పుడూ ఒకే output ఇస్తుంది.
- **Uniform** - keys ను buckets అంతటా సమానంగా distribute చేస్తుంది.
- **Fast** - key size కి అనుగుణంగా O(1) లో compute అవుతుంది.

### ఉదాహరణ

```
function simpleHash(key, size):
    total = 0
    for char in key:
        total += charCode(char)
    return total % size
```

ఇది ఉద్దేశపూర్వకంగా సరళంగా ఉంచబడింది - production hash functions అయిన MurmurHash3 లేదా xxHash మెరుగైన distribution కోసం bit-mixing ఉపయోగిస్తాయి.

## Collision Handling

Collisions తప్పనిసరి (pigeonhole principle). రెండు ప్రధాన strategies:

### Chaining

ప్రతి bucket లో ఒక linked list (లేదా చిన్న array) ఉంటుంది. Collide అయ్యే keys ను list కు append చేస్తారు.

```
function insert(table, key, value):
    idx = hash(key) % table.size
    for entry in table[idx]:
        if entry.key == key:
            entry.value = value
            return
    table[idx].append((key, value))
```

**ప్రయోజనాలు:** సరళం, మోస్తరు load లో బాగా పని చేస్తుంది.
**అప్రయోజనాలు:** Pointer chasing వల్ల cache-unfriendly.

### Open Addressing

అన్ని entries నేరుగా array లోనే ఉంటాయి. Collision అయినప్పుడు, తదుపరి ఖాళీ slot ను **probe** చేస్తారు.

- **Linear probing:** index+1, index+2, … తనిఖీ చేయండి.
- **Quadratic probing:** index+1², index+2², … తనిఖీ చేయండి.
- **Double hashing:** step size నిర్ణయించడానికి రెండవ hash function ఉపయోగించండి.

**ప్రయోజనాలు:** మెరుగైన cache performance, అదనపు pointers లేవు.
**అప్రయోజనాలు:** Clustering performance ను దెబ్బతీయవచ్చు; deletion కష్టం (tombstones అవసరం).

<FunFact>Python dictionaries custom probing sequence తో open addressing ఉపయోగిస్తాయి. CPython యొక్క dict implementation చాలా optimize చేయబడి ఉంది - ఆధునిక hardware పై సెకనుకు 5 కోట్ల కంటే ఎక్కువ lookups process చేస్తుంది.</FunFact>

## Load Factor మరియు Rehashing

**Load factor** α = entries సంఖ్య / buckets సంఖ్య. α పెరిగేకొద్దీ, collisions పెరుగుతాయి మరియు performance తగ్గుతుంది.

| Load Factor | అంచనా Lookups (chaining) |
|------------|--------------------------|
| 0.5 | ~1.25 |
| 0.75 | ~1.375 |
| 1.0 | ~1.5 |
| 2.0 | ~2.0 |

చాలా implementations α ఒక threshold (సాధారణంగా 0.75) దాటినప్పుడు **rehash** చేస్తాయి: పెద్ద array allocate చేసి, ప్రతి key యొక్క index తిరిగి compute చేసి, reinsert చేస్తాయి.

Rehashing O(n) అయినా అరుదుగా జరుగుతుంది, కాబట్టి insertion యొక్క **amortised** cost O(1) గానే ఉంటుంది.

<ThinkAboutIt>Rehashing table size ను double చేసి ప్రతి element ను తిరిగి insert చేస్తుంది. మీ AI system live request stream ను handle చేస్తుంటే, ఈ pause latency spike కు కారణమవుతుంది. Rehashing cost ను ఒకేసారి చేయకుండా అనేక insertions లో ఎలా విభజించవచ్చు?</ThinkAboutIt>

## AI లో వాస్తవ Applications

| Use Case | Hash Tables ఎలా సహాయపడతాయి |
|----------|------------------------------|
| **Embedding tables** | Token IDs ను O(1) లో dense vector representations కు map చేయడం |
| **Feature hashing** | High-cardinality categorical features ను fixed-size vector లోకి hash చేయడం ("hashing trick") |

| **Caching layers** | Input hash ద్వారా keyed precomputed model predictions ను store చేయడం |
| **Count-min sketch** | Multiple hash functions ఉపయోగించి streaming data లో approximate frequency counts |

## Bloom Filters: Probabilistic Membership

ఒక **Bloom filter** "ఈ element set లో ఉందా?" అనే ప్రశ్నకు సమాధానం ఇస్తుంది:
- **ఖచ్చితంగా లేదు** - element తప్పకుండా లేదు.
- **బహుశా ఉంది** - element ఉండే అవకాశం ఉంది (తక్కువ false-positive rate).

ఇది _k_ hash functions మరియు ఒక bit array ఉపయోగిస్తుంది. Insert చేసేటప్పుడు _k_ bits set అవుతాయి; lookup లో అన్ని _k_ bits set అయ్యాయో తనిఖీ చేస్తారు.

```
function bloomInsert(filter, item):
    for i in 1..k:
        idx = hashFunctions[i](item) % filter.size
        filter.bits[idx] = 1

function bloomContains(filter, item):
    for i in 1..k:
        idx = hashFunctions[i](item) % filter.size
        if filter.bits[idx] == 0: return false
    return true   // probably present
```

**AI లో ఉపయోగాలు:** ఒక URL ఇప్పటికే crawl అయిందో తనిఖీ చేయడం, spam filtering మరియు caching layers లో redundant database queries ను నివారించడం.

<FunFact>Google Chrome లక్షల known malicious URLs ఉన్న Bloom filter ఉపయోగిస్తుంది. ఏదైనా page load చేయడానికి ముందు, మీ browser filter ను తనిఖీ చేస్తుంది - అది "బహుశా ఉంది" అంటే, Chrome నిర్ధారించడానికి full list ను fetch చేస్తుంది.</FunFact>

## Sets vs Maps: ఏది ఎప్పుడు ఉపయోగించాలి

| Structure | ఏమి Store చేస్తుంది | ఎప్పుడు ఉపయోగించాలి |
|-----------|---------------------|----------------------|
| **Set** | Keys మాత్రమే | Membership checks కావాలి - "ఇది నేను ఇంతకు ముందు చూశానా?" |
| **Map** | Key–value pairs | ప్రతి key తో data associate చేయాలి - "ఈ key యొక్క value ఏమిటి?" |

రెండూ hash tables పై ఆధారపడి ఉంటాయి మరియు సగటున O(1) operations అందిస్తాయి. Deduplication, visited tracking మరియు uniqueness constraints కోసం **set** ఎంచుకోండి. Lookups, caches మరియు configuration stores కోసం **map** ఎంచుకోండి.

<ThinkAboutIt>ఒక AI content moderation system incoming messages ను 1 కోటి banned phrases blocklist తో తనిఖీ చేయాలి. మీరు hash set, Bloom filter లేదా రెండూ ఉపయోగిస్తారా? Memory, speed మరియు accuracy లో trade-offs ఏమిటి?</ThinkAboutIt>

## ముఖ్యమైన అంశాలు

1. Hash tables సగటున O(1) lookups అందిస్తాయి - caching, indexing మరియు feature engineering కు వెన్నెముక.
2. Collisions ను chaining లేదా open addressing ద్వారా manage చేస్తారు; load factor rehash ఎప్పుడు చేయాలో నిర్ణయిస్తుంది.
3. Bloom filters తక్కువ false-positive rate ను accept చేసి membership queries లో భారీ memory ఆదా చేస్తాయి.
4. Hashing internals అర్థం చేసుకోవడం ద్వారా ప్రతి AI workload కు సరైన tool - set, map లేదా probabilistic filter - ఎంచుకోవచ్చు.
