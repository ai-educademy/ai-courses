## Interview Prompt

*"100 million users కు personalised product suggestions అందించే e-commerce platform కోసం ఒక recommendation engine డిజైన్ చేయండి."*

ఈ ప్రశ్న మీ **machine learning pipelines** ను **scalable system design** తో కలపగల సామర్థ్యాన్ని పరీక్షిస్తుంది - senior-level interviews లో ఈ combination ఇప్పుడు చాలా common అయింది.

## Step 1: Requirements సేకరణ

**Functional requirements:**
- Homepage లో personalised "Recommended for You" (top-20 products)
- Product pages లో "Customers who bought this also bought"
- Regional activity ఆధారంగా "Trending in your area"
- Real-time updates: recommendations recent browsing ను minutes లో reflect చేయాలి

**Non-functional requirements:**
- 100ms (p99) లోపు recommendations serve చేయాలి
- Peak లో 50,000 requests per second handle చేయాలి
- 100 million users మరియు 10 million products support చేయాలి
- Graceful degradation: personalisation fail అయితే popular items చూపించాలి

<Callout>
ఎల్లప్పుడూ ఒక fallback strategy define చేయండి. "ML model fail అయితే, మనం gracefully popularity-based recommendations కు degrade అవుతాం" అని చెప్పడం interviewers కు చాలా నచ్చుతుంది - ఇది production thinking చూపిస్తుంది.
</Callout>

## Step 2: Recommendation విధానాలు

మూడు core techniques చాలా recommendation systems ను power చేస్తాయి:

**Collaborative filtering** - "మీలాంటి users కు ఈ items నచ్చాయి"
- User-item interaction matrices (views, purchases, ratings) analyse చేయండి
- Matrix factorisation ద్వారా similar users లేదా similar items కనుగొనండి
- తగినంత interaction data ఉంటే బాగా పని చేస్తుంది

**Content-based filtering** - "మీకు ఈ features ఉన్న items నచ్చాయి కాబట్టి"
- Product attributes (category, brand, price range, description) ఉపయోగించండి
- Transformer models తో product descriptions నుండి embeddings generate చేయండి
- Embedding space లో items మధ్య similarity compute చేయండి

**Hybrid approach** - బలమైన recommendations కోసం రెండు signals కలపండి
- చాలా production systems hybrid ఉపయోగిస్తాయి: personalisation కోసం collaborative signals, diversity మరియు cold-start coverage కోసం content signals

## Step 3: Recommendation Pipeline

Production recommendation systems సాధారణంగా **two-stage architecture** ఉపయోగిస్తాయి:

**Stage 1 - Candidate Generation (offline/near-real-time)**
- 10 million products ను ~1,000 candidates per user కు narrow చేయండి
- Lightweight models ఉపయోగించండి: collaborative filtering, embedding similarity
- Batch jobs (hourly/daily) లేదా streaming తో near-real-time గా run అవుతుంది

**Stage 2 - Ranking (real-time)**
- ~1,000 candidates ను richer model తో score చేయండి
- Context పరిగణించండి: time of day, device, recent activity, cart contents
- Top-20 ranked items return చేయండి
- 100ms latency budget లో execute కావాలి

## Step 4: Data Model మరియు Feature Store

Recommendations multiple data sources నుండి compute చేసిన rich features పై ఆధారపడతాయి:

**User features:** purchase history, browsing patterns, demographic segment, favourite categories
**Item features:** category, price, ratings, description embeddings, sales velocity
**Interaction features:** click-through rates, time spent viewing, add-to-cart rates

ఒక **feature store** (ఉదా. Feast, Tecton) అందిస్తుంది:
- **Offline features** - model training కోసం precomputed batch features
- **Online features** - real-time ranking కోసం low-latency feature retrieval
- **Consistency** - training మరియు serving లో same feature definitions

```
Feature Store Schema:
  user_features:    { user_id → [purchase_vector, category_prefs, avg_price, ...] }
  item_features:    { item_id → [embedding, category, price, popularity_score, ...] }
  interaction_features: { (user_id, item_id) → [clicks, views, last_interaction, ...] }
```

<FunFact>
Netflix అంచనా ప్రకారం దాని recommendation system subscriber churn తగ్గించడం ద్వారా కంపెనీకి సంవత్సరానికి $1 billion ఆదా చేస్తోంది. Relevant suggestions అందుకునే users తమ subscriptions cancel చేసే అవకాశం చాలా తక్కువ.
</FunFact>

## Step 5: Real-Time vs Batch Recommendations

| అంశం | Batch | Real-Time |
|------|-------|-----------|
| తాజాదనం | గంటల పాతది | సెకన్ల పాతది |
| Compute cost | తక్కువ (scheduled jobs) | ఎక్కువ (per-request inference) |
| Complexity | సరళమైన pipeline | Streaming infrastructure అవసరం |
| Use case | Homepage "for you" | "Because you just viewed X" |

**Practical hybrid approach:**
- ప్రతి కొన్ని గంటలకు batch లో candidate lists pre-compute చేయండి
- Latest session context తో real time లో re-rank చేయండి
- Events వచ్చినప్పుడు user features near-real-time update చేయడానికి streaming (Kafka) ఉపయోగించండి

<ThinkAboutIt>
మీరు Amazon లో ఒక product browse చేసినప్పుడు, recommendations దాదాపు వెంటనే update అవుతాయి. కానీ collaborative filtering models train చేయడానికి గంటలు పడుతుంది. Amazon batch model training మరియు real-time personalisation మధ్య ఈ gap ను ఎలా bridge చేస్తుందని మీరు అనుకుంటున్నారు?
</ThinkAboutIt>

## Step 6: A/B Testing Framework

Recommendation systems కు rigorous experimentation అవసరం:

- **Traffic splitting** - users లో ఒక percentage ను కొత్త model కు route చేయండి
- **Metrics** - click-through rate (CTR), conversion rate, revenue per session, diversity
- **Statistical significance** - reliable conclusions కోసం tests తగినంత సమయం run చేయండి
- **Guardrail metrics** - కొత్త models overall revenue లేదా engagement ను degrade చేయకుండా ensure చేయండి

ఒక typical setup **feature flag service** ఉపయోగిస్తుంది, users ను experiment buckets కు consistently assign చేయడానికి (same user ఎల్లప్పుడూ test సమయంలో same variant చూస్తారు).

## Step 7: Cold-Start Solutions

Cold-start problem కొత్త users మరియు కొత్త items రెండింటికీ వస్తుంది:

**కొత్త users:**
- మొదట popularity-based recommendations చూపించండి
- Onboarding questions అడగండి ("మీకు ఆసక్తి ఉన్న categories select చేయండి")
- Initial features గా demographic లేదా geographic signals ఉపయోగించండి
- 5-10 interactions తర్వాత personalised recommendations కు transition చేయండి

**కొత్త items:**
- Content-based features (description, category, price) తో items ను embedding space లో place చేయండి
- "New arrivals" section లో కొత్త items ను boost చేయండి
- Exploration/exploitation strategies run చేయండి (interaction data సేకరించడానికి కొత్త items ను users subset కు చూపించండి)

## Step 8: System ను Scale చేయడం

**50K requests per second కోసం architecture:**
- **API gateway** - rate limiting, authentication, request routing
- **Recommendation service** - stateless, load balancer వెనుక horizontally scaled
- **Feature store** - low-latency feature retrieval కోసం Redis cluster
- **Model serving** - memory లో pre-loaded models (TensorFlow Serving, Triton)
- **Pre-computed cache** - ప్రతి user top recommendations Redis cache, batch jobs ద్వారా refresh
- **Fallback** - cache miss మరియు model timeout అయితే popularity-based defaults return చేయండి

## ముఖ్యమైన విషయాలు

- Two-stage architecture ఉపయోగించండి: candidate generation (broad) → ranking (precise)
- Hybrid approaches (collaborative + content-based) single-method systems కంటే బాగా perform చేస్తాయి
- Feature stores model training మరియు real-time serving మధ్య consistency ensure చేస్తాయి
- A/B testing అవసరం - ML metrics మాత్రమే కాదు, business metrics కోసం optimise చేయండి
- Cold-start strategies కొత్త users మరియు items కు చెడ్డ experience నివారిస్తాయి
- Peak traffic handle చేయడానికి aggressively cache చేసి gracefully degrade అవ్వండి
