# Lanceer een Open-Source AI-Project — Jouw Meesterwerk

Dit is het — de laatste les. Alles wat je hebt geleerd komt hier samen: ML-pipelines, research-implementatie, system design en engineering-vakmanschap. Je gaat **een echt open-source AI-project opleveren** dat een reëel probleem oplost en voortleeft na deze cursus.

## 💡 Je Projectidee Kiezen

De beste open-source projecten komen voort uit **persoonlijke frustratie**. Bouw niet nóg een TODO-app met AI — zoek een gat:

- **Los je eigen irritatie op** — een tool die je mist in je dagelijkse workflow
- **Verbeter een bestaande tool** — vind een beperking van een populaire library en los die op
- **Implementeer een paper** — maak van een onderzoeksdoorbraak een bruikbare library
- **Overburg een kloof** — verbind twee ecosystemen die niet met elkaar communiceren

<Illustration src="/images/lessons/open-source-ai-project.svg" alt="Open-source projectlevenscyclus van idee via lancering tot communitygroei" caption="De open-source levenscyclus — lanceren is het halverwege-punt, niet de eindstreep." />

**Valideer voordat je bouwt**: doorzoek GitHub, PyPI en npm. Als er iets vergelijkbaars bestaat, heeft jouw project een duidelijk onderscheidend vermogen nodig — sneller, eenvoudigere API, betere documentatie of een nieuwe aanpak.

<FunFact>Hugging Face begon als chatbotbedrijf voor tieners. Toen ze hun Transformers-library als zijproject open-sourceden, werd het zo populair dat ze het hele bedrijf eromheen draaiden — nu meer dan $4,5 miljard waard.</FunFact>

## 📋 Projectplanning — MVP-scope

Definieer je **Minimum Viable Project** meedogenloos:

```markdown
## MVP-scope (Week 1-2)
- Kernfunctionaliteit: ÉÉN ding, goed gedaan
- CLI of Python API-interface
- 3 werkende voorbeelden
- Basis-README met installatie en quickstart

## V1.0-scope (Week 3-4)
- Uitgebreide documentatie
- Testdekking > 80%
- CI/CD-pipeline
- Gepubliceerd op PyPI/npm
- Demo gedeployed
```

**Het antipatroon**: maandenlang code perfectioneren die niemand heeft gezien. Lever vroeg op, itereer op basis van feedback.

<ThinkAboutIt>Denk aan de laatste drie tools die je installeerde. Waarom koos je ze boven alternatieven? Was het de README, het API-ontwerp, het aantal sterren, of iets anders? Pas die inzichten toe op je eigen project.</ThinkAboutIt>

## 🏗️ Repository-inrichting

Eerste indrukken tellen. Een professionele repository bevat: `src/`-indeling voor code, `tests/`, `examples/`, `docs/`, `.github/workflows/ci.yml`, plus rootbestanden — `LICENSE` (MIT of Apache 2.0), `README.md`, `CONTRIBUTING.md`, `CODE_OF_CONDUCT.md` en `pyproject.toml`.

### De README — Je Belangrijkste Bestand

Je README is een verkooppagina. Deze moet bevatten: **beschrijving in één zin**, **installatie** (`pip install my-project`), **quickstart** (werkende code in minder dan 10 regels), **kernfeatures** (opsommingstekens), **vergelijkingstabel** met alternatieven, en een **documentatielink**.

Een kopieerbare quickstart die meteen werkt, converteert bezoekers naar gebruikers. Al het andere is secundair aan die eerste succesvolle ervaring." />

## 📦 Publiceren op PyPI

Moderne Python-packaging met `pyproject.toml`:

```toml
[build-system]
requires = ["hatchling"]
build-backend = "hatchling.build"

[project]
name = "my-ai-project"
version = "0.1.0"
description = "One-line description of your project"
requires-python = ">=3.9"
license = "MIT"
dependencies = ["torch>=2.0", "numpy>=1.24"]

[project.urls]
Documentation = "https://my-ai-project.readthedocs.io"
Repository = "https://github.com/username/my-ai-project"
```

```bash
# Build en publiceer
pip install build twine
python -m build
twine upload dist/*
```

**Geautomatiseerde releases**: gebruik GitHub Actions om bij elke getagde release automatisch naar PyPI te publiceren — publiceer na de eerste keer nooit meer handmatig.

## 🎨 Demo's Maken

Een demo overtuigt sceptici en maakt ze tot gebruikers. Drie uitstekende platformen:

| Platform | Ideaal voor | Deployment |
|----------|------------|-----------|
| **Hugging Face Spaces** | ML-modeldemo's | Git push, gratis GPU-tier |
| **Streamlit** | Data-apps, dashboards | Streamlit Cloud, gratis tier |
| **Gradio** | Snelle ML-interfaces | Hugging Face Spaces of standalone |

```python
# Gradio-demo — 8 regels naar een deelbare interface
import gradio as gr
from my_ai_project import predict

demo = gr.Interface(
    fn=predict,
    inputs=gr.Textbox(label="Enter text"),
    outputs=gr.Label(label="Prediction"),
    title="My AI Project Demo"
)
demo.launch()
```

## 📖 Documentatie Schrijven

Volg het **Diátaxis-framework** — vier soorten documentatie: **Tutorials** (leergericht, stapsgewijs), **How-to guides** (taakgericht, praktisch), **Referentie** (volledige API-docs) en **Uitleg** (conceptueel begrip). Gebruik **MkDocs** of **Docusaurus** om sites te genereren uit Markdown. Gratis deployen op GitHub Pages of Read the Docs.

## 📣 Je Project Promoten

Bouwen is de helft van de strijd. Mensen je project laten gebruiken is de andere helft:

1. **Hacker News** — "Show HN: [beschrijving]" post op lanceerddag
2. **Reddit** — r/MachineLearning, r/Python, relevante subreddits
3. **Twitter/X** — thread die het probleem en je oplossing uitlegt, met een GIF-demo
4. **Product Hunt** — als je project een webinterface heeft
5. **Blogpost** — gedetailleerd technisch artikel op je persoonlijke site of Medium/Dev.to

**Timing is belangrijk**: lanceer op dinsdag t/m donderdag ochtend (GMT). Vermijd weekenden en feestdagen.

<FunFact>De makers van FastAPI, LangChain en Streamlit noemen een goed getimede "Show HN"-post op Hacker News allemaal als de grootste aanjager van hun initiële adoptiegolf.</FunFact>

<ThinkAboutIt>Je project krijgt 500 GitHub-sterren in de eerste week maar nul issues of PR's van externe bijdragers. Is dit een succes? Wat zou je anders doen om passieve stargazers om te zetten in actieve bijdragers?</ThinkAboutIt>

## 🔧 Je Project Onderhouden

Duurzaamheid is belangrijker dan viraliteit:

- **Label issues** als `good-first-issue`, `help-wanted`, `bug`, `enhancement`
- **Reageer op issues binnen 48 uur** — zelfs een "Bedankt, ik ga ernaar kijken" bouwt vertrouwen op
- **Schrijf een CONTRIBUTING.md** met setup-instructies, codeerstandaarden en PR-richtlijnen
- **Gebruik semantic versioning** — breaking changes krijgen een major-versienummer
- **Automatiseer alles** — CI, releases, changelog-generatie, dependency-updates

## 💰 Verdienmodellen

Open source betekent niet voor altijd gratis werken. Duurzame modellen: **GitHub Sponsors** (lage inspanning, terugkerende donaties), **consulting** (betaalde setup en training), **SaaS-wrapper** (gehoste versie met enterprise-features — meeste inspanning maar duurzaamst), **dual licensing** (open-source + commercieel) en **bedrijfssponsoring** (prioriteitsfeatures voor betalende bedrijven).

## 🎯 Checklist voor Je Meesterwerk

Voordat je het als opgeleverd beschouwt:

- [ ] Kernfunctionaliteit werkt en is getest
- [ ] README heeft installatie, quickstart en feature-overzicht
- [ ] Gepubliceerd op PyPI of npm
- [ ] Live demo op Hugging Face Spaces of Streamlit
- [ ] CI/CD-pipeline draait tests bij elke PR
- [ ] Documentatie behandelt tutorials en API-referentie
- [ ] LICENSE, CONTRIBUTING.md en CODE_OF_CONDUCT.md aanwezig
- [ ] Gelanceerd op minstens twee platformen (HN, Reddit, Twitter)

**Dit project ÍS je portfolio.** Als een interviewer vraagt "Wat heb je gebouwd?", heb je een live, gedocumenteerd, gepubliceerd antwoord.
