# End-to-End ML Pipeline - Raw Data నుండి Deployed Model వరకు

చాలా tutorials `model.fit()` వద్ద ఆగిపోతాయి. Production లో అది పని లో 20% మాత్రమే. ఈ పాఠం **మొత్తం ML lifecycle** ద్వారా నడిపిస్తుంది - Spotify లో recommendation engines, Stripe లో fraud detection, Google లో search ranking ని power చేసే అదే pipeline.

## 🗺️ ML Lifecycle ఒక చూపులో

<Illustration src="/images/lessons/end-to-end-ml-pipeline.svg" alt="Data ingestion నుండి monitoring వరకు ML pipeline దశల రేఖాచిత్రం" caption="Production ML pipeline యొక్క ఎనిమిది దశలు - ప్రతి ఒక్కటి సంభావ్య failure point." />

ప్రతి production ML system ఈ దశల ద్వారా వెళ్తుంది:

1. **Data Collection** - APIs, databases, event streams, web scraping
2. **Data Cleaning** - nulls handling, deduplication, type coercion
3. **Feature Engineering** - raw signals ని model-ready features గా మార్చడం
4. **Model Selection** - మీ సమస్యకు match అయ్యే architectures ఎంచుకోవడం
5. **Training** - GPU/CPU orchestration, hyperparameter tuning
6. **Evaluation** - offline metrics, A/B testing, fairness audits
7. **Deployment** - serving infrastructure, latency budgets
8. **Monitoring** - drift detection, alerting, retraining trigger

## 📦 Data Collection మరియు Versioning

Raw data మీ పునాది. దీన్ని source code లాగా treat చేయండి - **అంతా version చేయండి**.

```bash
# Initialise DVC in your repo
dvc init
dvc remote add -d storage s3://my-ml-data-bucket

# Track a dataset
dvc add data/transactions.csv
git add data/transactions.csv.dvc .gitignore
git commit -m "Track raw transaction dataset v1"
dvc push
```

**DVC** (Data Version Control) ద్వారా మీరు ఏ historical dataset నైనా `git checkout` చేయవచ్చు. Git tags తో కలిపి, ఏ సమయం నుండైనా ఏ experiment నైనా reproduce చేయవచ్చు.

<FunFact>Netflix రోజుకు 1.5 ట్రిలియన్ events ని తమ data pipelines ద్వారా process చేస్తుంది - ప్రతి సెకను సుమారు 17 మిలియన్ events వారి recommendation models కి feed అవుతాయి.</FunFact>

## 🛠️ Feature Engineering - పోటీలు గెలిచే చోటు

Algorithms కంటే features ఎక్కువ ముఖ్యం. అద్భుతమైన features ఉన్న gradient-boosted tree, పేలవమైన features ఉన్న deep neural network ని ఓడిస్తుంది.

```python
import pandas as pd

df = pd.read_csv("data/transactions.csv", parse_dates=["timestamp"])

# Temporal features
df["hour"] = df["timestamp"].dt.hour
df["is_weekend"] = df["timestamp"].dt.dayofweek >= 5

# Aggregation features
user_stats = df.groupby("user_id")["amount"].agg(["mean", "std", "count"])
user_stats.columns = ["user_avg_amount", "user_std_amount", "user_txn_count"]
df = df.merge(user_stats, on="user_id")

# Ratio features
df["amount_vs_avg"] = df["amount"] / df["user_avg_amount"]
```

## 🧪 MLflow తో Experiment Tracking

మీరు ఏమి try చేశారో ఎప్పుడూ track కోల్పోకండి. MLflow parameters, metrics, artefacts ని automatically log చేస్తుంది.

```python
import mlflow
from sklearn.ensemble import GradientBoostingClassifier
from sklearn.metrics import f1_score

mlflow.set_experiment("fraud-detection-v2")

with mlflow.start_run(run_name="gbm-baseline"):
    params = {"n_estimators": 500, "max_depth": 6, "learning_rate": 0.05}
    mlflow.log_params(params)

    model = GradientBoostingClassifier(**params)
    model.fit(X_train, y_train)

    score = f1_score(y_test, model.predict(X_test))
    mlflow.log_metric("f1_score", score)
    mlflow.sklearn.log_model(model, "model")
```

MLflow UI లో runs compare చేసి, ఉత్తమ model ని staging → production lifecycle tags తో **Model Registry** కి promote చేయండి.

## 🚀 Pipeline as Code - Orchestration

Manual steps విఫలమవుతాయి. Orchestration frameworks వాడి మీ pipeline ని code లో encode చేయండి:

| Tool | దేనికి ఉత్తమం | కీలక బలం |
|------|-------------|----------|
| **Kubeflow Pipelines** | Kubernetes-native ML | ఇప్పటికే ఉన్న K8s clusters పై scale అవుతుంది |
| **Apache Airflow** | Complex DAG scheduling | Mature ecosystem, విస్తృత adoption |
| **Prefect** | Modern Python workflows | Pythonic API, అద్భుతమైన error handling |

## 🌐 Model Serving మరియు Deployment

మీ model కి API అవసరం. రెండు ప్రధాన patterns:

**REST API** - సరళం, HTTP-based, moderate throughput కి గొప్పది:
```python
# FastAPI model server
from fastapi import FastAPI
import joblib

app = FastAPI()
model = joblib.load("model.pkl")

@app.post("/predict")
async def predict(features: dict):
    prediction = model.predict([list(features.values())])
    return {"prediction": int(prediction[0])}
```

**gRPC** - binary protocol, REST కంటే 2–10× వేగం, internal microservices మరియు high-throughput scenarios కి ideal.

### Canary Deployments

కొత్త model కి 100% traffic ఎప్పుడూ పంపకండి. **Canary releases** వాడండి: 5% → 25% → 50% → 100%, ప్రతి step లో error rates monitor చేస్తూ. Metrics degrade అయితే instantly rollback చేయండి.

## 📊 Monitoring మరియు Feedback Loop

Deployment finish line కాదు - starting gun. ఇవి monitor చేయండి:

- **Data drift** - training data నుండి input distributions shift అవడం
- **Concept drift** - features మరియు target మధ్య relationship మారడం
- **Performance degradation** - accuracy/latency metrics decline అవడం

**Evidently AI**, **WhyLabs**, **Arize** లాంటి tools drift dashboards అందిస్తాయి. Drift thresholds దాటినప్పుడు, fresh production data తో automatic retraining trigger చేయండి - feedback loop close చేస్తూ.
