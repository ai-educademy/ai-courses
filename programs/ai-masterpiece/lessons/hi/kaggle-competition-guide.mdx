# Kaggle Competition Guide — Kaggle पर कैसे Compete करें (और सीखें)

Kaggle सिर्फ एक competition platform नहीं — यह दुनिया की सबसे बड़ी applied ML classroom है। 15 million से ज़्यादा data scientists इसे skills sharpen करने, portfolio बनाने, और jobs पाने के लिए use करते हैं। यह lesson आपको सिखाएगा कि effectively compete कैसे करें और हर competition से maximum learning कैसे निकालें।

| Type | उदाहरण | Typical Winning Approach |
|------|---------|------------------------|
| **Tabular** | House prices, fraud detection | Gradient boosting (XGBoost/LightGBM) + heavy feature engineering |

## 🔍 EDA — आपके पहले 48 घंटे

Data समझे बिना कभी modelling शुरू न करें। एक disciplined EDA workflow:

```python
import pandas as pd
import matplotlib.pyplot as plt

df = pd.read_csv("train.csv")

# Shape और types — जानें कि आप किसके साथ काम कर रहे हैं
print(f"Shape: {df.shape}")
print(f"Missing values:\n{df.isnull().sum().sort_values(ascending=False).head(10)}")

# Target distribution — क्या यह balanced है?
df["target"].value_counts(normalize=True).plot(kind="bar")
plt.title("Target Distribution")
plt.show()

# Correlations — quick signal खोजें
correlations = df.select_dtypes(include="number").corr()["target"].sort_values()
print(correlations.head(10))  # Strongest negative correlations
print(correlations.tail(10))  # Strongest positive correlations
```

**Leakage check करें** — ऐसे features जो prediction time पर exist नहीं करेंगे। यह beginners की नंबर एक गलती है। अगर कोई feature target के साथ suspicious तरीके से correlate करता है, तो खुशी मनाने से पहले investigate करें।

## 🛠️ Feature Engineering जो जिताती है

Top competitors अपना 70% समय features पर लगाते हैं, model tuning पर नहीं। आज़माई हुई techniques:

**Aggregation features** — अलग-अलग granularities पर group statistics:
```python
for col in ["category", "store_id", "day_of_week"]:
    stats = df.groupby(col)["sales"].agg(["mean", "std", "median"])
    stats.columns = [f"{col}_sales_{s}" for s in ["mean", "std", "median"]]
    df = df.merge(stats, on=col, how="left")
```

**Target encoding** — categories को smoothed target means से replace करें (leakage रोकने के लिए fold-based encoding use करें)।

**Lag features** — time series के लिए, पिछले values सोने जैसे हैं:
```python
for lag in [1, 7, 14, 28]:
    df[f"sales_lag_{lag}"] = df.groupby("store_id")["sales"].shift(lag)
```

## ✅ Cross-Validation — अपने Local Score पर भरोसा करें

आपका local CV score public leaderboard से ज़्यादा मायने रखता है। एक robust validation strategy:

```python
from sklearn.model_selection import StratifiedKFold

skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
oof_predictions = np.zeros(len(X))

for fold, (train_idx, val_idx) in enumerate(skf.split(X, y)):
    X_train, X_val = X.iloc[train_idx], X.iloc[val_idx]
    y_train, y_val = y.iloc[train_idx], y.iloc[val_idx]

    model = lgb.LGBMClassifier(n_estimators=1000, learning_rate=0.05)
    model.fit(X_train, y_train, eval_set=[(X_val, y_val)],
              callbacks=[lgb.early_stopping(50)])
    oof_predictions[val_idx] = model.predict_proba(X_val)[:, 1]

print(f"OOF AUC: {roc_auc_score(y, oof_predictions):.5f}")
```

**Golden rule**: अगर local CV और public LB disagree करें, तो CV पर भरोसा करें। Public leaderboard test data का सिर्फ एक हिस्सा use करता है — इस पर overfit होना एक जाल है।

## 🎭 Ensemble Methods — आखिरी Push

लगभग हर winning solution ensembles use करती है। तीन key techniques:

**Bagging** — same model को अलग-अलग data subsets पर train करें, predictions average करें। Variance कम करता है।

**Stacking** — diverse base models के out-of-fold predictions पर एक meta-model train करें:
```python
# Level 1: diverse base models OOF predictions बनाते हैं
base_preds = np.column_stack([lgbm_oof, xgb_oof, catboost_oof, nn_oof])

# Level 2: logistic regression optimal combination सीखता है
from sklearn.linear_model import LogisticRegression
meta = LogisticRegression()
meta.fit(base_preds, y_train)
```

**Blending** — model predictions का weighted average। Stacking से simple लेकिन effective:
```python
final = 0.4 * lgbm_pred + 0.35 * xgb_pred + 0.25 * catboost_pred
```

## 📓 बेहतरीन Kaggle Notebooks लिखना

Public notebooks आपकी reputation बनाते हैं। Medal-worthy notebook में होना चाहिए:

1. **साफ कहानी** — अपनी reasoning explain करें, सिर्फ code नहीं
2. **Reproducibility** — random seeds set करें, library versions document करें
3. **Visualisations** — insights दिखाने वाले plots, सजावट नहीं
4. **ईमानदार results** — जो काम नहीं किया उसे भी share करें
