# Engineering Portfolio

AI युग में, आपका portfolio आपका सबसे powerful career asset है। ये तब बोलता है जब आप room में नहीं होते, तब काम करता है जब आप सो रहे होते हैं, और एक साथ हज़ारों hiring managers तक पहुँच सकता है।

एक polished CV आपको pile में डालता है। एक strong portfolio आपको **pile से बाहर** निकालता है।

<Illustration src="/images/lessons/engineering-portfolio.svg" alt="GitHub, blog, और personal website को एक साथ connected दिखाती developer profile" caption="आपका portfolio एक ecosystem है - हर piece दूसरे को reinforce करता है" />

## Portfolios CVs से बेहतर क्यों हैं

CV किसी को बताता है कि आपने क्या किया claim करते हैं। Portfolio इसे **prove** करता है।

AI और software engineering में hiring managers एक CV पर average छह seconds बिताते हैं। लेकिन जब वो किसी अच्छी GitHub profile या personal site पर click करते हैं, तो minutes बिताते हैं - कभी-कभी और भी ज़्यादा। वो extra attention ही rejection email और first-round interview के बीच का फ़र्क़ है।

<Callout>
अपने portfolio को एक product समझें और hiring manager को अपना user। हर element friction कम करे और trust बनाए।
</Callout>

## GitHub: आपकी Public Workshop

आपकी GitHub profile वो पहली जगह है जहाँ technical recruiters देखते हैं। इसे count कराएँ।

### Profile README
अपने username के नाम पर एक repository में `README.md` बनाएँ। ये आपकी profile के top पर दिखता है। इसमें शामिल करें:

- आप कौन हैं और किस पर focus करते हैं - एक-line summary।
- आपकी strongest skills highlight करते दो-तीन bullet points।
- आपके best projects, blog, और LinkedIn के links।

### Pinned Repositories
आप छह repositories तक pin कर सकते हैं। ऐसे projects चुनें जो **range और depth** दिखाएँ:

- एक end-to-end ML project जिसमें deployment हो।
- एक backend या systems project जो engineering rigour demonstrate करे।
- एक open-source contribution (meaningful changes वाला fork भी चलेगा)।

### Contribution Graph
Consistency intensity से ज़्यादा matter करती है। Green squares का steady graph reliability signal करता है। Sporadic bursts की बजाय छोटे, regular commits का aim रखें।

## Project Showcase: क्या बनाएँ

Hiring manager की नज़र में सभी projects बराबर नहीं हैं। सबसे अच्छे portfolio projects में तीन चीज़ें common होती हैं:

1. **वो एक real problem solve करते हैं** - भले ही छोटी हो।
2. **वो deployed और accessible हैं** - live link हर बार README से जीतता है।
3. **उनमें एक write-up है** - जो आपके decisions, trade-offs, और सीखे गए lessons explain करे।

ऐसे projects से बचें जो clearly tutorial follow-alongs हैं। अगर हज़ार और लोगों ने वही चीज़ उसी code से बनाई है, तो ये आपको differentiate नहीं करती।

<ThinkAboutIt>
अपने current projects देखें। कौन सा आप interview में demo करके सबसे ज़्यादा proud feel करेंगे? अगर कोई भी excite नहीं करता, तो ये signal है कि कुछ नया बनाने का वक़्त है।
</ThinkAboutIt>

## Technical Blog Posts लिखें

Blog सबसे high-leverage portfolio item है जिसे ज़्यादातर engineers ignore करते हैं।

**ये क्यों काम करता है:**
- ये communication skills demonstrate करता है, जो engineering में scarce हैं।
- ये recruiters और collaborators से inbound interest attract करता है।
- ये आपकी ख़ुद की समझ deepen करता है - जो सच में नहीं जानते वो explain नहीं कर सकते।

**किस बारे में लिखें:**
- एक technical decision जो आपने लिया और क्यों।
- एक bug जिसे debug करने में घंटे लगे और क्या सीखा।
- दो tools या approaches का comparison जिन्हें आपने evaluate किया।

**कहाँ publish करें:**
- **Dev.to** या **Hashnode** discoverability के लिए।
- **अपनी ख़ुद की site** long-term ownership के लिए।
- Maximum reach के लिए दोनों पर cross-post करें।

<FunFact>
Stack Overflow के co-founder Jeff Atwood अपने blog "Coding Horror" को अपनी career का single biggest factor मानते हैं। उन्होंने सालों तक consistently लिखा जो अब तक के सबसे important developer platforms में से एक बनने से पहले था।
</FunFact>

## Engineers के लिए LinkedIn Optimisation

बहुत से engineers LinkedIn neglect करते हैं, लेकिन recruiters वहीं रहते हैं। कुछ high-impact tweaks:

- **Headline**: "Software Engineer at X" skip करें। बदले में: "ML Engineer | Building AI-powered search systems | Open to opportunities"।
- **About section**: तीन छोटे paragraphs लिखें - आप क्या करते हैं, क्या बनाया है, और क्या ढूँढ रहे हैं।
- **Featured section**: अपनी best blog post, project demo, या conference talk pin करें।
- **Skills**: Vague terms (problem-solving, teamwork) की बजाय specific technologies list करें (PyTorch, Kubernetes, FastAPI)।

## Personal Website: क्या शामिल करें

आपकी personal site आपका home base है। इसे simple और fast-loading रखें।

**शामिल करें:**
- Professional photo के साथ एक short bio।
- Screenshots या demos के साथ top तीन projects के links।
- आपका blog या उसका link।
- Contact information या एक simple form।

**छोड़ दें:**
- Fancy animations जो page slow करें।
- Percentage bars वाला skills section - इनका कोई मतलब नहीं।
- हर project जो आपने कभी बनाया हो। Ruthlessly curate करें।

<FunFact>
Google की hiring research में पाया गया कि personal website वाले candidates को बिना website वालों से 27% ज़्यादा callback मिलता है, experience level और qualifications को control करने के बाद भी।
</FunFact>

## Portfolio Review Checklist

Portfolio complete मानने से पहले, ये list check करें:

- [ ] GitHub profile में README है, छह pinned repos हैं, और consistent contribution graph है।
- [ ] कम से कम एक project live link के साथ deployed है।
- [ ] हर pinned project में clear README है जिसमें purpose, setup instructions, और screenshot या demo हो।
- [ ] कम से कम एक technical blog post publish की है।
- [ ] LinkedIn headline, about section, और featured content optimised हैं।
- [ ] Personal website तीन seconds से कम में load होती है और mobile पर काम करती है।
- [ ] एक non-technical दोस्त आपकी site से समझ सके कि आप क्या करते हैं।

<ThinkAboutIt>
अपना portfolio link tech से बाहर किसी को भेजें और पूछें: "क्या तुम बता सकते हो कि मैं क्या करता हूँ और उसमें अच्छा हूँ?" उनका जवाब वो blind spots reveal करेगा जो आप ख़ुद नहीं देख सकते।
</ThinkAboutIt>
