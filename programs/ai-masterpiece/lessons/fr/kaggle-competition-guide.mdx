# Guide des compétitions Kaggle - Comment concourir (et apprendre) sur Kaggle

Kaggle n'est pas qu'une plateforme de compétitions - c'est la plus grande salle de cours de ML appliqué au monde. Plus de 15 millions de data scientists l'utilisent pour affûter leurs compétences, construire leur portfolio et décrocher des emplois. Cette leçon vous apprend à concourir efficacement et à tirer le maximum de chaque compétition.

## 🎯 Pourquoi Kaggle compte pour votre carrière

L'expérience Kaggle signale quelque chose que les CV ne peuvent pas : **vous savez livrer des modèles qui fonctionnent sur des données réelles et bruitées**. Les recruteurs chez Google, Meta et les meilleures startups recrutent activement sur les classements Kaggle. Même sans gagner, un profil solide avec des notebooks bien documentés démontre une compétence pratique.

## 📊 Types de compétitions et stratégies gagnantes

| Type | Exemple | Approche gagnante typique |
|------|---------|--------------------------|
| **Tabulaire** | Prix immobiliers, détection de fraude | Gradient boosting (XGBoost/LightGBM) + feature engineering intensif |
| **Vision par ordinateur** | Classification d'images, segmentation | CNN pré-entraînés (EfficientNet, ConvNeXt) + augmentation |
| **NLP** | Analyse de sentiment, question answering | Transformers fine-tunés (DeBERTa, RoBERTa) |
| **Simulation** | IA de jeu, optimisation | Reinforcement learning + heuristiques métier |

## 🔍 EDA - Vos 48 premières heures

Ne commencez jamais la modélisation avant d'avoir compris vos données. Un workflow EDA discipliné :

```python
import pandas as pd
import matplotlib.pyplot as plt

df = pd.read_csv("train.csv")

# Dimensions et types - savoir avec quoi on travaille
print(f"Shape: {df.shape}")
print(f"Missing values:\n{df.isnull().sum().sort_values(ascending=False).head(10)}")

# Distribution de la cible - est-elle équilibrée ?
df["target"].value_counts(normalize=True).plot(kind="bar")
plt.title("Target Distribution")
plt.show()

# Corrélations - trouver du signal rapidement
correlations = df.select_dtypes(include="number").corr()["target"].sort_values()
print(correlations.head(10))  # Strongest negative correlations
print(correlations.tail(10))  # Strongest positive correlations
```

**Vérifiez les fuites de données** - des features qui n'existeraient pas au moment de la prédiction. C'est l'erreur numéro un des débutants. Si une feature corrèle anormalement bien avec la cible, enquêtez avant de vous réjouir.

## 🛠️ Le feature engineering qui fait gagner

Les meilleurs compétiteurs consacrent 70 % de leur temps aux features, pas au tuning de modèles. Techniques éprouvées :

**Features d'agrégation** - statistiques de groupe à différentes granularités :
```python
for col in ["category", "store_id", "day_of_week"]:
    stats = df.groupby(col)["sales"].agg(["mean", "std", "median"])
    stats.columns = [f"{col}_sales_{s}" for s in ["mean", "std", "median"]]
    df = df.merge(stats, on=col, how="left")
```

**Target encoding** - remplacer les catégories par des moyennes lissées de la cible (utilisez un encodage par fold pour éviter les fuites).

**Features de décalage** - pour les séries temporelles, les valeurs passées valent de l'or :
```python
for lag in [1, 7, 14, 28]:
    df[f"sales_lag_{lag}"] = df.groupby("store_id")["sales"].shift(lag)
```

## ✅ Validation croisée - Fiez-vous à votre score local

Votre score de CV local compte plus que le classement public. Une stratégie de validation robuste :

```python
from sklearn.model_selection import StratifiedKFold

skf = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
oof_predictions = np.zeros(len(X))

for fold, (train_idx, val_idx) in enumerate(skf.split(X, y)):
    X_train, X_val = X.iloc[train_idx], X.iloc[val_idx]
    y_train, y_val = y.iloc[train_idx], y.iloc[val_idx]

    model = lgb.LGBMClassifier(n_estimators=1000, learning_rate=0.05)
    model.fit(X_train, y_train, eval_set=[(X_val, y_val)],
              callbacks=[lgb.early_stopping(50)])
    oof_predictions[val_idx] = model.predict_proba(X_val)[:, 1]

print(f"OOF AUC: {roc_auc_score(y, oof_predictions):.5f}")
```

**Règle d'or** : si votre CV local et le classement public divergent, fiez-vous à votre CV. Le classement public n'utilise qu'une fraction des données de test - s'y sur-ajuster est un piège.

<ThinkAboutIt>Vous êtes 15e au classement public mais votre CV local suggère que votre modèle est sur-ajusté. Soumettez-vous votre meilleur score du classement ou votre meilleur score CV pour la soumission finale ? Pourquoi ?</ThinkAboutIt>

## 🎭 Méthodes d'ensemble - Le sprint final

Quasiment chaque solution gagnante utilise des ensembles. Trois techniques clés :

**Bagging** - entraîner le même modèle sur différents sous-ensembles de données, moyenner les prédictions. Réduit la variance.

**Stacking** - entraîner un méta-modèle sur les prédictions out-of-fold de modèles de base diversifiés :
```python
# Niveau 1 : des modèles de base diversifiés produisent des prédictions OOF
base_preds = np.column_stack([lgbm_oof, xgb_oof, catboost_oof, nn_oof])

# Niveau 2 : la régression logistique apprend la combinaison optimale
from sklearn.linear_model import LogisticRegression
meta = LogisticRegression()
meta.fit(base_preds, y_train)
```

**Blending** - moyenne pondérée des prédictions de modèles. Plus simple que le stacking mais efficace :
```python
final = 0.4 * lgbm_pred + 0.35 * xgb_pred + 0.25 * catboost_pred
```

En les moyennant, les erreurs individuelles s'annulent tandis que les signaux corrects se renforcent - réduisant la variance globale sans augmenter le biais." />

## 📓 Écrire de bons notebooks Kaggle

Les notebooks publics construisent votre réputation. Un notebook digne d'une médaille inclut :

1. **Un récit clair** - expliquez votre raisonnement, pas seulement votre code
2. **La reproductibilité** - fixez les random seeds, documentez les versions des bibliothèques
3. **Des visualisations** - des graphiques qui révèlent des insights, pas de la simple décoration
4. **Des résultats honnêtes** - partagez ce qui n'a pas marché autant que ce qui a marché

## 🎯 Points clés à retenir

- EDA d'abord, modélisation ensuite - vérifiez toujours les fuites de données
- Le feature engineering apporte plus de gains que le tuning d'hyperparamètres
- Fiez-vous à votre CV local plutôt qu'au classement public
- Ensemblez des modèles diversifiés pour le sprint final
- Documentez votre travail dans des notebooks publics - c'est votre portfolio
